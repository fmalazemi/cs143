02 · From Python

You Already Wrote Functions

In CS101 you called functions and wrote your own with def. C++ uses the same idea, but you must state the types.

Python

def add(a, b):
    return a + b

print(add(2, 3))   # 5

C++

int add(int a, int b) {
    return a + b;
}
// in main:
cout << add(2, 3);   // 5

New parts: the return type (int) and a type for each parameter.

03 · The Idea

Why Use Functions?

Without functions

  • The same code is copied many times.
  • A fix must be made in every copy.
  • One long main is hard to read.

With functions

  • Write the logic once, use it anywhere.
  • Fix a bug in one place.
  • Each function has a clear, named job.

Think of a function as a small machine: you give it inputs, it does a job, and it may hand back a result.

04 · The Parts

Anatomy of a Function

int
Return type
square
Name
(int x)
Parameters
int square(int x) {     // header: type, name, parameter
    return x * x;       // body: the work + a result
}
  • Return type: the kind of value the function gives back (here, an int).
  • Name: how you call it later.
  • Parameters: the inputs the function receives, each with a type.
  • return: sends a value back to whoever called the function.
05 · Call and Return

Calling a Function

When you call a function, the program jumps into it, runs its body, then comes back with the returned value.

So square(5) becomes 25, and that value is printed.

06 · Inputs

Parameters and Arguments

#include <iostream>
using namespace std;

// a and b are PARAMETERS (named inputs)
int add(int a, int b) {
    return a + b;
}

int main() {
    // 2 and 3 are ARGUMENTS (real values)
    cout << add(2, 3) << endl;   // 5
    cout << add(10, 7) << endl;  // 17
    return 0;
}
Expected Output
5 17
  • Parameter: the name inside the function (a, b).
  • Argument: the value you pass when calling (2, 3).
  • The count and types must match the function header.
07 · Two Kinds

Returning a Value vs. void

Returns a value

int twice(int x) {
    return x * 2;   // hands back a number
}
// use it:  int y = twice(6);  // 12

Returns nothing (void)

void greet() {
    cout << "Hello!" << endl;
}
// use it:  greet();   // just does a job

Rule: a function with a return type other than void must use return to give back a value. A void function does a job but returns no value.

08 · Declare Before Use

Function Prototypes

#include <iostream>
using namespace std;

int add(int a, int b);   // prototype (a promise)

int main() {
    cout << add(2, 3) << endl;
    return 0;
}

// full definition comes later
int add(int a, int b) {
    return a + b;
}

C++ reads your file top to bottom. It must know a function before you call it.

  • A prototype tells C++ the function's shape: return type, name, and parameter types, ending with ;.
  • Then you can call it in main and write the full body afterwards.
  • Shortcut: if you define the whole function above main, you do not need a prototype.
Worked Example 1

Largest of Two (returns int)

#include <iostream>
using namespace std;

int maxOfTwo(int a, int b) {
    if (a > b) return a;   // give back the bigger one
    else       return b;
}

int main() {
    cout << maxOfTwo(8, 3) << endl;   // 8
    cout << maxOfTwo(4, 9) << endl;   // 9
    return 0;
}
Expected Output
8 9

What it does: It compares the two inputs and returns whichever is larger.

Note: a function can have more than one return; the first one reached ends the function.

Worked Example 2

Print a Row of Stars (void)

#include <iostream>
using namespace std;

// does a job, returns nothing
void printStars(int n) {
    for (int i = 0; i < n; i++) {
        cout << "*";
    }
    cout << endl;
}

int main() {
    printStars(3);
    printStars(6);
    return 0;
}
Expected Output
*** ******

What it does: It prints n stars on a line. Because it only prints, its return type is void.

Reuse: one function, called twice with different inputs.

Worked Example 3

Is It Even? (returns bool)

#include <iostream>
using namespace std;

// returns true or false
bool isEven(int n) {
    return (n % 2 == 0);
}

int main() {
    int x;
    cin >> x;
    if (isEven(x))
        cout << x << " is even" << endl;
    else
        cout << x << " is odd" << endl;
    return 0;
}
If user types 7
7 is odd

What it does: The function returns a bool (true/false). The result is used straight inside an if.

Clean code: the test has a clear name — isEven — instead of bare arithmetic.

09 · How Inputs Travel

Pass by Value vs. by Reference

swap with reference (&)

void swap(int &a, int &b) {
    int t = a; a = b; b = t;
}

By value (no &): the function gets a copy; the original is safe.

By reference (int &a): the function shares the original, so it can change it — needed for swap.

10 · Where Variables Live

Local Scope

#include <iostream>
using namespace std;

void demo() {
    int x = 99;        // x lives only inside demo
    cout << x << endl;
}

int main() {
    int x = 5;         // a different x
    demo();            // prints 99
    cout << x << endl;  // prints 5
    return 0;
}
Expected Output
99 5
  • A variable declared inside a function is local — it exists only there.
  • The two x variables are completely separate.
  • When a function ends, its local variables disappear.
11 · Be Careful

Common Mistakes

  • No return type
    every function needs one (int, void, …)
  • Missing return
    a non-void function must return a value
  • Called before declared
    define it first or add a prototype
  • Wrong argument count/type
    they must match the parameters
  • Forgetting ( )
    greet; does nothing; write greet();
  • Expecting by-value to change the original
    use a reference & for that

6 · Swap with references

#include <iostream>
using namespace std;

void swap(int &a, int &b) {
    int temp = a;   // hold a
    a = b;          // a takes b's value
    b = temp;       // b takes old a
}

int main() {
    int x = 1, y = 2;
    swap(x, y);
    cout << x << " " << y << endl;
    return 0;
}
Expected Output
2 1

Why it works: the parameters are references, so a and b are the same boxes as x and y. The swap is seen in main.

Without the &, the function would only swap copies and main would still print 1 2.

15 · Wrap Up

Summary

  • A function is a named block of code: return type, name, parameters, body.
  • Calling a function jumps into it and comes back with the returned value.
  • A void function does a job; other functions must return a value.
  • Declare a function before you call it — define it first or use a prototype.
  • By value copies the input; by reference (&) shares it so changes are kept.
  • Variables declared inside a function are local to it.
