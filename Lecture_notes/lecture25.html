Binary Search

Finding a value in a sorted array, fast

The Problem

Searching a Long List

Given an array and a target value, find the index of the target — or report that it is not there.

Linear search

Check every element from left to right.

for (int i = 0; i < n; i++)
    if (arr[i] == target)
        return i;
return -1;

Works on any array. Slow: up to n checks.

Binary search

Look at the middle. Because the array is sorted, one comparison throws away half the array.

Needs a sorted array. Fast: about log₂ n checks.

This is why we sort: sorting once makes every later search enormously faster.

The Payoff

Halving Beats Walking

Each comparison in binary search removes half of what is left. The number of steps grows very slowly.

Array size nLinear search (worst)Binary search (worst)
1010 checks4 checks
1,0001,000 checks10 checks
1,000,0001,000,000 checks20 checks

Doubling the array adds only one extra step. A million items are searched in twenty comparisons.

Each step halves the problem: 1000000 → 500000 → 250000 → … → 1

The Rule You Must Not Break

The Array Must Be Sorted

Binary search decides which half to keep by comparing with the middle value. If the array is not sorted, that decision is meaningless.

Unsorted — it fails

arr = 16 2 91 5 23 search 5 → -1 (not found)

But 5 is in the array, at index 3! The search compared with 91, decided to look left, and missed it.

Sorted — it works

arr = 2 5 16 23 91 search 5 → 1 (found)

Sort first, then search. sort(arr, arr + n);

Remember: a wrong answer is worse than a slow one.

The Idea

Three Variables, Three Cases

We keep the part of the array that could still contain the target, using two indices.

The variables

  • low — first index still in play.
  • high — last index still in play.
  • mid — the middle of that range.

At each step, compare arr[mid] with the target

  • Equal → found it. Return mid.
  • Too small → the target is to the right: low = mid + 1
  • Too large → the target is to the left: high = mid - 1

Repeat while low <= high. If they cross, the target is not present.

The Decision

One Comparison, Half the Array Gone

The Syntax

Finding the Middle

int mid = low + (high - low) / 2;

Why not (low + high) / 2?

For very large arrays, low + high can be bigger than int can hold. The result would be wrong.

The safe form

low + (high - low) / 2 gives the same answer and never overflows.

Integer division rounds down: with low = 0 and high = 9, mid is 4, not 4.5.

The Trace

Searching for 23

Worked Example 1

Binary Search With a Loop

#include <iostream>
using namespace std;

// returns the index of target, or -1 if absent
int binarySearch(int arr[], int n, int target) {
    int low = 0;
    int high = n - 1;

    while (low <= high) {          // still a range left
        int mid = low + (high - low) / 2;

        if (arr[mid] == target)
            return mid;            // found
        else if (arr[mid] < target)
            low = mid + 1;         // keep the right half
        else
            high = mid - 1;        // keep the left half
    }
    return -1;                     // not found
}

int main() {
    int arr[10] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
    cout << binarySearch(arr, 10, 23) << endl;
    return 0;
}
Expected Output
5

What it does: 23 sits at index 5. The loop found it in 3 comparisons.

  • low <= high means the range is not empty.
  • mid + 1 and mid - 1 exclude the value we just tested.
Worked Example 2

When the Target Is Absent

int arr[10] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};

int result = binarySearch(arr, 10, 10);   // 10 is not there

if (result == -1)
    cout << "Not found" << endl;
else
    cout << "Found at index " << result << endl;
The range shrinks to nothing

Expected Output
Not found

What it does: after four comparisons, high becomes smaller than low. The range is empty, so the target cannot be there.

Returning -1 is the usual way to say "absent", because -1 is never a valid index.

Worked Example 3

Binary Search With Recursion

int binarySearch(int arr[], int low, int high, int target) {
    // base case 1: empty range
    if (low > high)
        return -1;

    int mid = low + (high - low) / 2;

    // base case 2: found it
    if (arr[mid] == target)
        return mid;

    if (arr[mid] < target)                    // search right
        return binarySearch(arr, mid + 1, high, target);
    else                                      // search left
        return binarySearch(arr, low, mid - 1, target);
}

int main() {
    int arr[10] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
    cout << binarySearch(arr, 0, 9, 23) << endl;
    return 0;
}
Expected Output
5

What it does: the same algorithm, written as a recursion. The shrinking range is passed as low and high.

Two base cases

  • low > high → empty range → -1
  • arr[mid] == target → success

Only one recursive call runs, so a loop is easily preferred.

The Growth

Why Twenty Steps Are Enough

Each comparison halves the range. Starting from a million, the range reaches one after about twenty halvings.

Be Careful

Common Mistakes

  • Searching an unsorted array
    the answer is simply wrong
  • Infinite loop
    high = mid; instead of high = mid - 1; never shrinks the range
  • Using low < high
    a one-element range is skipped; use low <= high
  • Overflow in mid
    prefer low + (high - low) / 2
  • Starting high = n
    the last index is n - 1
  • Forgetting the not-found case
    always return -1 after the loop

The infinite-loop test: after every step, the range must get strictly smaller.

2 · Why sorted?

Because the comparison with arr[mid] is what tells us which half to discard; without order, that half may still contain the target.

3 · Safe mid

int mid = low + (high - low) / 2;

It gives the same value as (low + high) / 2 but never lets the sum grow too large for an int, so it cannot overflow.

Solutions (4–6)

Exercise Answers

4 · high = mid

The range stops shrinking once high equals low + 1, so mid keeps taking the same value: an infinite loop. The tested element must be excluded.

6 · One million values

About 20 comparisons, because each step halves the range and 2²⁰ is just over a million.

5 · Complete function

int binarySearch(int arr[], int n, int target) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}
Wrap Up

Summary

  • Binary search finds a value in a sorted array by repeatedly halving the range.
  • The array must be sorted, or the answer is wrong.
  • Keep low and high; test arr[mid]; discard half each time.
  • Use mid + 1 and mid - 1 so the range always shrinks.
  • Loop while low <= high; return -1 when they cross.
  • A million values need only about 20 comparisons.

Sort once, then search almost instantly — forever after.
