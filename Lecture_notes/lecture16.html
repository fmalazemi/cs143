Function Overloading & Default Arguments

One name for many cases, and parameters with built-in values

Part A · The Idea

One Name, Several Versions

Function overloading means giving several functions the same name, as long as their parameters are different. The compiler chooses the right one from the arguments you pass.

Without overloading

printInt(5);
printDouble(3.14);
printString("hi");

A different name for each type — harder to remember.

With overloading

print(5);
print(3.14);
print("hi");

One simple name; the right version is chosen for you.

The benefit: simpler, more natural names for related tasks.

Part A · How It Works

The Compiler Matches by Arguments

Each call is matched to the version whose parameters fit the arguments — by their number and types.

Worked Example 1

Overloading by Parameter Type

void print(int x)    { cout << "int: " << x << endl; }
void print(double x) { cout << "double: " << x << endl; }
void print(string x) { cout << "string: " << x << endl; }

int main() {
    print(5);        // matches print(int)
    print(3.14);     // matches print(double)
    print("hi");     // matches print(string)
    return 0;
}
Expected Output
int: 5 double: 3.14 string: hi

What it does: three functions share the name print but differ by parameter type. The argument's type decides which runs.

Worked Example 2

Overloading by Number of Parameters

int add(int a, int b)        { return a + b; }
int add(int a, int b, int c) { return a + b + c; }

int main() {
    cout << add(2, 3) << endl;      // 5
    cout << add(2, 3, 4) << endl;   // 9
    return 0;
}
Expected Output
5 9

What it does: both functions are named add. The count of arguments decides which one runs.

Overloads may differ by type, by number, or by both.

Part A · An Important Rule

Return Type Does Not Count

Overloads must differ in their parameters. The return type alone is not enough.

int f(); and double f(); clash — same name, same (empty) parameters.

Worked Example 3

Overloading Methods in a Class

#include <iostream>
using namespace std;

class Calculator {
public:
    int add(int a, int b) {
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
};

int main() {
    Calculator c;
    cout << c.add(2, 3) << endl;       // 5
    cout << c.add(2.5, 1.5) << endl;   // 4
    return 0;
}
Expected Output
5 4

What it does: methods can be overloaded just like free functions. The integer call uses the int version; the decimal call uses the double one.

Box() and Box(int,int) were constructor overloading — the same idea.

Part B · Default Arguments

Parameters With a Built-in Value

A default argument gives a parameter a value to use when the caller leaves it out. One function then serves several call forms.

Worked Example 4

Using Default Arguments

#include <iostream>
using namespace std;

void greet(string name = "friend",
           string word = "Hello") {
    cout << word << ", " << name << "!" << endl;
}

int main() {
    greet();                 // both defaults
    greet("Sara");           // name given
    greet("Ali", "Welcome"); // both given
    return 0;
}
Expected Output
Hello, friend! Hello, Sara! Welcome, Ali!

What it does: one function handles three call styles. Any argument you leave out uses its default value.

Part B · The Rule

Defaults Must Be on the Right

// OK: the default is the last parameter
void f(int a, int b = 10);

// OK: the last two have defaults
void g(int a, int b = 2, int c = 3);

// ERROR: a default cannot come
// before a normal parameter
void h(int a = 1, int b);   // wrong
  • Default arguments must be the trailing (right-most) parameters.
  • Once one parameter has a default, every parameter to its right must too.
  • Give the default once (normally in the declaration).

Why? When you skip arguments, C++ fills in from the right.

Part B · Choosing

Overloading vs. Default Arguments

OverloadingDefault arguments
How many functionsseveral, one per casejust one
Bodies can differyes, fullyno, one shared body
Best whencases need different codecases differ only by a value

Caution — do not let them clash

void f(int a);
void f(int a, int b = 0);
f(5);   // ERROR: ambiguous — matches both
Part B · In Classes

Defaults in Methods and Constructors

class Box {
private:
    int w, h;
public:
    // one constructor, three ways to call it
    Box(int width = 1, int height = 1) {
        w = width;
        h = height;
    }
    int area() { return w * h; }
};

int main() {
    Box a;          // 1 x 1
    Box b(5);       // 5 x 1
    Box c(5, 3);    // 5 x 3
    cout << a.area() << " " << b.area()
         << " " << c.area();   // 1 5 15
}
Expected Output
1 5 15

What it does: a single constructor with defaults replaces several overloaded constructors. Missing arguments fall back to 1.

Be Careful

Common Mistakes

Overloading

  • Only the return type differs
    int f(); vs double f(); clash
  • Identical parameter lists
    same parameters is not allowed
  • Renaming a parameter only
    f(int a) vs f(int b) is the same

Default arguments

  • Default not on the right
    defaults must be the trailing parameters
  • Giving the default twice
    state it once, not in both places
  • Clashing with an overload
    a call may then match two versions

Remember: overloads differ by parameters; defaults fill in from the right.

Solutions (1–3)

Exercise Answers

1 · maxOf (overloaded)

int maxOf(int a, int b) {
    return (a > b) ? a : b;
}
double maxOf(double a, double b) {
    return (a > b) ? a : b;
}
// maxOf(3, 7) -> 7 ; maxOf(2.5, 1.1) -> 2.5

2 · greet (default)

void greet(string name = "friend") {
    cout << "Hello, " << name << endl;
}
// greet();       -> Hello, friend
// greet("Sara"); -> Hello, Sara

3 · area (overloaded)

int area(int side) {       // square
    return side * side;
}
int area(int w, int h) {   // rectangle
    return w * h;
}
// area(4) -> 16 ; area(4, 3) -> 12
Solutions (4–6)

Exercise Answers

4 · power (default exp)

int power(int base, int exp = 2) {
    int result = 1;
    for (int i = 0; i < exp; i++)
        result *= base;
    return result;
}
// power(5)    -> 25  (exp = 2)
// power(2, 3) -> 8

5 · Why it fails

A default argument must be a trailing parameter. In h(int a = 1, int b), b has no default but comes after one that does, so C++ could not fill arguments from the right.

6 · Box with defaults

class Box {
private:
    int w, h;
public:
    Box(int width = 1, int height = 1) {
        w = width; h = height;
    }
    int area() { return w * h; }
};

int main() {
    Box a;       // 1 x 1  -> 1
    Box b(4);    // 4 x 1  -> 4
    Box c(4, 3); // 4 x 3  -> 12
}
Wrap Up

Summary

  • Overloading: several functions share one name, differing by the number or types of parameters.
  • The compiler picks the version that fits the arguments; the return type alone does not count.
  • Methods and constructors can be overloaded in the same way.
  • Default arguments give a parameter a built-in value used when the caller omits it.
  • Defaults must be the trailing parameters, and stated once.
  • Use overloading when cases need different code; use defaults when they differ only by a value.

Both let one clear name serve many related calls.
