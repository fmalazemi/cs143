The Big Picture

What Is the STL?

The Standard Template Library is a part of standard C++. It gives you ready-made tools, so you do not have to build them yourself.

Containers

Ready-made ways to store collections of data — like a resizable array, a set, or a dictionary.

Algorithms

Ready-made operations that work on those collections — like sorting, reversing, and searching.

Today: first the containers (Part A), then the algorithms (Part B). Each tool needs its own #include.

Part A

Containers

Ways to store a collection of values: vector, array, set, and map.

Part A · Containers

Four Containers at a Glance

Worked Example 1 · vector

vector — a Resizable Array

#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> v;       // empty, can grow
    v.push_back(10);     // add to the end
    v.push_back(20);
    v.push_back(30);

    cout << v[0] << endl;       // 10
    cout << v.size() << endl;    // 3
    for (int x : v) cout << x << " ";  // 10 20 30
    return 0;
}
Expected Output
10 3 10 20 30
  • push_back adds an item to the end.
  • v[i] reads an item; v.size() gives the count.
  • Unlike a plain array, a vector grows as needed.

Coming from Python? A vector is like a list.

Part A · Containers

array — a Fixed-Size Array

#include <iostream>
#include <array>
using namespace std;

int main() {
    array<int, 5> a = {1, 2, 3, 4, 5};

    cout << a[2] << endl;     // 3
    cout << a.size() << endl;  // 5
    return 0;
}
  • array<int, 5> holds exactly 5 ints — a fixed size.
  • It knows its own length: a.size().
  • Safer and tidier than a plain C-style array.

Use array when the size never changes; use vector when it might grow.

Part A · Containers

set — Unique, Sorted Values

#include <iostream>
#include <set>
using namespace std;

int main() {
    set<int> s;
    s.insert(3);
    s.insert(1);
    s.insert(3);   // duplicate is ignored

    for (int x : s) cout << x << " ";  // 1 3
    cout << endl;
    cout << s.count(2);  // 0  (not present)
    return 0;
}
Expected Output
1 3 0
  • A set keeps only unique values, always sorted.
  • insert adds; a repeat is ignored.
  • count(x) is 1 if present, 0 if not.

Coming from Python? A set is like a set.

Worked Example 2 · map

map — Key to Value

#include <iostream>
#include <map>
using namespace std;

int main() {
    map<string, int> age;
    age["Sara"] = 20;     // key -> value
    age["Ali"]  = 22;

    cout << age["Sara"] << endl;   // 20

    for (auto p : age)            // each pair
        cout << p.first << ": " << p.second << endl;
    return 0;
}
Expected Output
20 Ali: 22 Sara: 20
  • A map stores key → value pairs.
  • p.first is the key, p.second is the value.
  • Keys are unique and kept in sorted order.

Coming from Python? A map is like a dict.

Part A · Choosing

Which Container?

ContainerHoldsUse when…
vectoran ordered list that can growyou add or remove items over time
arraya fixed number of itemsthe size is known and never changes
setunique values, sortedyou need no duplicates
mapkey → value pairsyou look things up by a key

Rough Python guide: vector ≈ list, set ≈ set, map ≈ dict.

Part B

Algorithms

Ready-made operations that work on a container: sort, reverse, and find.

Part B · Algorithms

Algorithms Work on a Range

Algorithms live in <algorithm>. They act on a range given by begin() (the first item) and end() (just past the last).

Part B · Algorithms

sort — Put Items in Order

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    vector<int> v = {3, 1, 2};

    sort(v.begin(), v.end());   // ascending

    for (int x : v) cout << x << " ";  // 1 2 3
    return 0;
}
Expected Output
1 2 3
  • sort arranges the range in increasing order.
  • It changes the container in place.
  • For largest-first, use sort(v.begin(), v.end(), greater<int>()).
Part B · Algorithms

reverse — Flip the Order

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    vector<int> v = {1, 2, 3};

    reverse(v.begin(), v.end());

    for (int x : v) cout << x << " ";  // 3 2 1
    return 0;
}
Expected Output
3 2 1

What it does: reverse turns the range back to front. The first item becomes the last.

Like sort, it changes the container in place.

Part B · Algorithms

find — Search for a Value

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    vector<int> v = {10, 20, 30};

    auto it = find(v.begin(), v.end(), 20);

    if (it != v.end())
        cout << "found";
    else
        cout << "not found";
    return 0;
}
Expected Output
found
  • find returns a marker (an iterator) to the item.
  • If the value is missing, it returns end().
  • So compare with end() to test "found or not".

auto lets the compiler work out the type for you.

Worked Example 3 · together

Sort, Then Search

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    vector<int> v = {40, 10, 30, 20};

    sort(v.begin(), v.end());        // 10 20 30 40
    for (int x : v) cout << x << " ";
    cout << endl;

    auto it = find(v.begin(), v.end(), 30);
    cout << (it != v.end() ? "30 is here" : "no 30");
    return 0;
}
Expected Output
10 20 30 40 30 is here

What it does: the same vector is first sorted, then searched. The container and the algorithms work together smoothly.

Be Careful

Common Mistakes

  • Missing the include
    each tool needs its header (<vector>, <algorithm>, …)
  • Forgetting begin/end
    sort(v) is wrong; use sort(v.begin(), v.end())
  • Expecting find to return an index
    it returns an iterator, not a number
  • push_back on an array
    array has a fixed size; only vector grows
  • Expecting duplicates in a set
    repeats are silently ignored
  • Reading a vector out of range
    v[i] beyond size() is unsafe

Tip: remember algorithms take a range, written as v.begin(), v.end().

Wrap Up

Summary

  • The STL gives ready-made containers and algorithms in standard C++.
  • vector — a resizable array; array — a fixed-size array.
  • set — unique, sorted values; map — key → value pairs.
  • Algorithms in <algorithm> work on the range begin() … end().
  • sort orders items, reverse flips them, find searches for a value.
  • find returns end() when the value is not present.

The STL lets you do more with less code — and fewer mistakes.
