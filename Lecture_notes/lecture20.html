The Concept

What Is Inheritance?

Inheritance lets a new class receive the attributes and methods of an existing class for free, and then add or change what it needs.

Without inheritance

  • Copy the same data and methods into every new class.
  • Fix a bug once — it stays broken in all the copies.

With inheritance

  • Write shared code once in the parent (base) class.
  • Each child (derived) class adds only what is different.

The is-a test: a child class should be an is-a version of the parent. A Rectangle is-a Shape.

The Picture

A Class Hierarchy

The parent (base) class sits at the top. Each child (derived) class inherits everything and adds its own details.

The Syntax

Declaring a Derived Class

// parent class
class Shape {
public:
    string colour;
    void describe() {
        cout << "Colour: " << colour << endl;
    }
};

// child class — inherits from Shape
class Rectangle : public Shape {
public:
    double width;
    double height;
    double area() { return width * height; }
};
  • class Rectangle : public Shape means Rectangle inherits from Shape.
  • The word public keeps the parent's public members public in the child.
  • Rectangle gets colour and describe() for free.
  • Rectangle adds its own width, height, and area().
What the Child Gets

Inherited vs. Own Members

Worked Example 1

Using Inherited Members

#include <iostream>
using namespace std;

class Shape {
public:
    string colour;
    void describe() {
        cout << "Colour: " << colour << endl;
    }
};

class Rectangle : public Shape {
public:
    double width, height;
    double area() { return width * height; }
};

int main() {
    Rectangle r;
    r.colour = "blue";        // inherited from Shape
    r.width  = 5;
    r.height = 3;
    r.describe();             // inherited method
    cout << r.area() << endl; // 15
    return 0;
}
Expected Output
Colour: blue 15

What it does: Rectangle uses colour and describe() even though they are defined only in Shape. We wrote the colour logic once — every shape subclass gets it.

Access in Inheritance

protected — Shared With Children

class Shape {
protected:               // accessible in children
    string colour;
public:
    Shape(string c) : colour(c) { }
    void describe() {
        cout << "Colour: " << colour << endl;
    }
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(string c, double r)
        : Shape(c), radius(r) { }
    double area() {
        return 3.14159 * radius * radius;
    }
    void info() {
        // can use colour because it is protected
        cout << colour << " circle, area = "
             << area() << endl;
    }
};

Use protected when children need direct access, but outside code should not.

Setting Up

Calling the Parent Constructor

When a child object is created, the parent constructor runs first. Call it by name in the child's initializer list.

Worked Example 2

Constructors in a Hierarchy

#include <iostream>
using namespace std;

class Shape {
protected:
    string colour;
public:
    Shape(string c) : colour(c) { }
    void describe() {
        cout << "Colour: " << colour << endl;
    }
};

class Rectangle : public Shape {
private:
    double width, height;
public:
    Rectangle(string c, double w, double h)
        : Shape(c), width(w), height(h) { }
    double area() { return width * height; }
    void info() {
        describe();
        cout << "Area: " << area() << endl;
    }
};

int main() {
    Rectangle r("red", 4, 5);
    r.info();
    return 0;
}
Expected Output
Colour: red Area: 20

What it does: Rectangle's constructor passes "red" up to Shape's constructor, which sets colour. The child then handles its own width and height.

Changing Behaviour

Overriding a Parent Method

class Shape {
public:
    string colour;
    Shape(string c) : colour(c) { }
    double area() { return 0; }         // base version
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(string c, double w, double h)
        : Shape(c), w(w), h(h) { }
    double area() { return w * h; }     // overrides Shape::area
};

class Circle : public Shape {
    double r;
public:
    Circle(string c, double r)
        : Shape(c), r(r) { }
    double area() {                     // overrides Shape::area
        return 3.14159 * r * r;
    }
};
  • If the child defines a method with the same name and parameters, it overrides the parent's version.
  • Each shape gives its own correct formula for area().

This is a stepping stone to polymorphism — the next lecture.

Worked Example 3

Each Shape Computes Its Own Area

#include <iostream>
using namespace std;

class Shape {
public:
    string colour;
    Shape(string c) : colour(c) { }
    double area() { return 0; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(string c, double w, double h)
        : Shape(c), w(w), h(h) { }
    double area() { return w * h; }
};

class Circle : public Shape {
    double r;
public:
    Circle(string c, double r)
        : Shape(c), r(r) { }
    double area() { return 3.14159 * r * r; }
};

int main() {
    Rectangle rect("blue", 4, 5);
    Circle    circ("red",  3);
    cout << rect.colour << " rect area = " << rect.area() << endl;
    cout << circ.colour << " circle area = " << circ.area() << endl;
    return 0;
}
Expected Output
blue rect area = 20 red circle area = 28.2743

What it does: both shapes inherit colour from Shape, but each overrides area() with its own correct formula.

Using Both Versions

Calling the Parent's Method Explicitly

class Shape {
public:
    string colour;
    Shape(string c) : colour(c) { }
    void describe() {
        cout << "Colour: " << colour << endl;
    }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(string c, double w, double h)
        : Shape(c), w(w), h(h) { }
    void describe() {
        Shape::describe();       // call the parent version first
        cout << "Size: " << w << " x " << h << endl;
    }
};
If we call r.describe()
Colour: green Size: 5 x 3

What it does: Shape::describe() calls the parent's version explicitly. The child extends the parent's behaviour rather than replacing it entirely.

Going Deeper

Multi-Level Inheritance

Vehicle → Car → ElectricCar

class Vehicle {
public:
    int speed;
};
class Car : public Vehicle {
public:
    string brand;
};
class ElectricCar : public Car {
public:
    int batteryKm;
};

The is-a chain

  • An ElectricCar is-a Car.
  • A Car is-a Vehicle.
  • An ElectricCar is-a Vehicle too.
  • An ElectricCar object has speed, brand, and batteryKm.

Each level adds more specific detail.

Be Careful

Common Mistakes

  • Accessing private parent data
    a child cannot touch private members — use protected
  • Forgetting to call the parent constructor
    the parent's fields need the parent's constructor to set them
  • Missing public in the colon
    class Rect : Shape defaults to private, breaking access
  • Confusing override with overload
    override keeps the same signature; overload changes it
  • Using is-a when it should be has-a
    a Car has-a Engine — not is-a
  • Very deep hierarchies
    three or more levels quickly become hard to follow

Quick test: say "A [child] is a [parent]." If it sounds natural, inheritance is the right tool.

class Employee : public Person {
    double salary;
public:
    Employee(string n, int a, double s)
        : Person(n, a), salary(s) { }
    void introduce() {
        Person::introduce();           // parent first
        cout << "Salary: " << salary << endl;

    int batteryKm;
public:
    ElectricCar(int s, string b, int km)
        : Car(s, b), batteryKm(km) { }
    void print() {
        Car::print();
        cout << "Range: " << batteryKm << " km" << endl;
    }

6 · Missing public

class Rectangle : Shape defaults to private inheritance, which makes the parent's public members private inside Rectangle. Outside code can no longer call them.

Always write class Rectangle : public Shape for normal inheritance.

Wrap Up

Summary

  • Inheritance lets a child class receive the attributes and methods of a parent class for free.
  • Write class Rectangle : public Shape to derive Rectangle from Shape.
  • protected members are visible in child classes but hidden from outside code.
  • Call the parent constructor from the child's initializer list: : Shape(c).
  • A child can override a parent method by defining one with the same name and parameters.
  • Call the parent's version explicitly with Shape::describe().
