Class Templates

Writing one class that works for any type

Instructor: Dr. Fawaz Alazemi

Builds on: classes, and the STL (vector, map)

Today's Goals

Learning Objectives

By the end of this lecture, you will be able to:

  • Explain what a class template is and why it helps.
  • Write a template with template <typename T>.
  • Create objects such as Box<int> and Box<string>.
  • Use a template with more than one type parameter.
  • Recognise that vector and map are class templates.

The Problem

The Same Class, Again and Again

class IntBox {
    int value;
public:
    IntBox(int v) : value(v) { }
    int get() { return value; }
};

class DoubleBox {           // almost identical...
    double value;
public:
    DoubleBox(double v) : value(v) { }
    double get() { return value; }
};
// ...and a StringBox, and a CharBox, and...
  • These classes differ only in one type.
  • Copying the whole class for each type is repetitive and error-prone.
  • Fix a bug in one and you must fix it in all the copies.

We want to write the class once and let the type vary.

The Idea

A Template Is a Class With a Blank for the Type

A class template uses a placeholder type — usually called T — instead of a fixed type. You fill in the real type later.

template <typename T>
class Box {
    T value;              // T stands in for a real type
public:
    Box(T v) : value(v) { }
    T get() { return value; }
};

One definition now serves int, double, string, and any other type.

The Picture

One Template, Many Versions

When you use Box<int>, the compiler makes a version of the class with T replaced by int.

The Syntax

How to Write a Class Template

template <typename T>
class Box { ... };
  • template <typename T> goes right before the class.
  • T is a name you choose — it stands for whatever type is used.
  • Inside the class, use T wherever you would normally write a type.
  • typename and class mean the same here: template <class T> also works.
What Really Happens

T Becomes the Type You Choose

Using a Template

Name the Type in Angle Brackets

Box<int>    a(5);        // a Box that holds an int
Box<string> b("hi");     // a Box that holds a string
Box<double> c(3.14);     // a Box that holds a double

cout << a.get() << endl;   // 5
cout << b.get() << endl;   // hi
  • Write the real type in angle brackets: Box<int>.
  • That type replaces T everywhere for this object.
  • Box<int> and Box<string> are different types.

You have seen these brackets before — in vector<int>.

Worked Example 1

One Box for Every Type

#include <iostream>
#include <string>
using namespace std;

template <typename T>
class Box {
    T value;
public:
    Box(T v) : value(v) { }
    T get() { return value; }
    void set(T v) { value = v; }
};

int main() {
    Box<int>    a(5);
    Box<string> b("hello");

    cout << a.get() << endl;   // 5
    cout << b.get() << endl;   // hello
    return 0;
}
Expected Output
5 hello

What it does: the single Box template serves both an int and a string. The compiler builds the right version for each.

Good to Know

Defining Members Outside the Class

template <typename T>
class Box {
    T value;
public:
    Box(T v);
    T get();
};

// each outside definition repeats the template line
template <typename T>
Box<T>::Box(T v) : value(v) { }

template <typename T>
T Box<T>::get() {
    return value;
}
  • Every member defined outside the class needs its own template <typename T> line.
  • The class name becomes Box<T> before the ::.
  • For beginners, defining members inside the class is simpler.

Templates usually live entirely in a header file.

More Than One Type

Templates With Several Type Parameters

template <typename K, typename V>
class Pair {
    K key;
    V value;
public:
    Pair(K k, V v) : key(k), value(v) { }
    K getKey()   { return key; }
    V getValue() { return value; }
};
  • List each type parameter, separated by commas: <typename K, typename V>.
  • Use them independently inside the class.
  • Create one by naming both types: Pair<string, int>.

This is the same idea behind map<string, int>.

Worked Example 2

A Pair of Two Types

#include <iostream>
#include <string>
using namespace std;

template <typename K, typename V>
class Pair {
    K key;
    V value;
public:
    Pair(K k, V v) : key(k), value(v) { }
    K getKey()   { return key; }
    V getValue() { return value; }
};

int main() {
    Pair<string, int> p("age", 30);
    cout << p.getKey() << ": "
         << p.getValue() << endl;   // age: 30
    return 0;
}
Expected Output
age: 30

What it does: Pair holds a key of one type and a value of another. Here K is string and V is int.

You Already Use These

The STL Is Built From Templates

Every STL container is a class template. The angle brackets you have been writing are template type arguments.

You writeWhat it means
vector<int>the vector template with T = int
vector<string>the same template with T = string
map<string, int>a template with two type parameters

The takeaway: you have used templates all along — now you can write your own.

Worked Example 3

A Generic Stack

#include <iostream>
#include <vector>
using namespace std;

template <typename T>
class Stack {
    vector<T> items;
public:
    void push(T x) { items.push_back(x); }
    T    top()     { return items.back(); }
    void pop()     { items.pop_back(); }
    bool empty()   { return items.empty(); }
};

int main() {
    Stack<int> s;
    s.push(1);
    s.push(2);
    cout << s.top() << endl;   // 2
    s.pop();
    cout << s.top() << endl;   // 1
    return 0;
}
Expected Output
2 1

What it does: Stack<T> works for any type. Change Stack<int> to Stack<string> and it stores words instead — with no other changes.

A Short Comparison

How Python Differs

GoalPythonC++
Hold any typeworks automaticallyuse a template
Name the typenot neededBox<int>
Type checkingat run timeat compile time

Python is dynamically typed, so a container or function already accepts any type — no templates needed. C++ is statically typed: templates give that same flexibility while the compiler still checks types for you.

Templates bring Python-like flexibility without giving up compile-time safety.

Be Careful

Common Mistakes

  • Forgetting the type
    Box a(5); — write Box<int> a(5);
  • Missing the template line
    each outside member needs template <typename T>
  • Mixing versions
    Box<int> and Box<double> are different types
  • Using an unsupported operation
    if the code uses <, the type T must support <
  • Splitting into a .cpp file
    keep template definitions in the header
  • Confusing << with < >
    angle brackets name a type, not a shift

Rule: a template becomes a real class only when you name a type in < >.

Wrap Up

Summary

  • A class template writes one class that works for any type, using a placeholder T.
  • Put template <typename T> before the class; use T as a type inside.
  • Create objects by naming the type: Box<int>, Box<string>.
  • Templates can take several type parameters, like Pair<K, V>.
  • The STL containers — vector, map, and more — are all class templates.
  • Rule: a template becomes a real class only when you name a type in < >.

Write it once, use it for every type.
