Constructors & Destructors

Setting up an object at birth, and cleaning up at the end

The Problem

Setting Data by Hand Is Risky

Without a constructor

Student s;
s.setName("Sara");
s.setId(101);
// easy to forget one of these!

You must set every field after creating the object. Forgetting one leaves it half-ready.

With a constructor

Student s("Sara", 101);
// fully ready in one step

The object is set up the moment it is created — nothing is forgotten.

A constructor makes sure an object always starts in a valid, complete state.

The Idea

What Is a Constructor?

A constructor is a special method that runs automatically when an object is created. It usually copies the given values into the object's attributes.

The Syntax

Writing a Constructor

class Point {
private:
    int x, y;

public:
    // constructor: same name as the class, NO return type
    Point(int a, int b) {
        x = a;
        y = b;
    }
    void print() { cout << x << "," << y; }
};

int main() {
    Point p(3, 5);   // the constructor runs here
    p.print();        // 3,5
}
  • A constructor has the same name as the class.
  • It has no return type — not even void.
  • It runs automatically when you write Point p(3, 5);

A cleaner form exists: Point(int a, int b) : x(a), y(b) {} — an initializer list.

Worked Example 1

A Student With a Constructor

#include <iostream>
using namespace std;

class Student {
private:
    string name;
    int id;
public:
    Student(string n, int i) {   // constructor
        name = n;
        id = i;
    }
    void print() {
        cout << name << " " << id << endl;
    }
};

int main() {
    Student s("Sara", 101);
    s.print();   // Sara 101
    return 0;
}
Expected Output
Sara 101

What it does: the constructor copies "Sara" and 101 into the private attributes the moment the object is created.

The object is complete and valid right away.

No Arguments

The Default Constructor

class Point {
private:
    int x, y;
public:
    Point() {        // default: takes no arguments
        x = 0;
        y = 0;
    }
    void print() { cout << x << "," << y; }
};

int main() {
    Point p;     // no parentheses needed
    p.print();   // 0,0
}
  • A default constructor takes no arguments.
  • It gives the object sensible starting values.
  • Create the object with just Point p; — no values.

If you write no constructor at all, C++ supplies a simple empty one for you.

More Than One

Several Constructors

class Box {
private:
    int w, h;
public:
    Box() {              // default
        w = 1; h = 1;
    }
    Box(int a, int b) {  // parameterized
        w = a; h = b;
    }
    int area() { return w * h; }
};

int main() {
    Box b1;          // uses Box()
    Box b2(4, 3);    // uses Box(int,int)
    cout << b1.area() << " " << b2.area();  // 1 12
}
  • A class can have more than one constructor.
  • They differ by their parameters.
  • C++ picks the one that matches how you create the object.

This is the same idea as overloaded functions, applied to constructors.

The Other Half

What Is a Destructor?

class Demo {
public:
    Demo() {           // constructor
        cout << "created" << endl;
    }
    ~Demo() {          // destructor (note the ~)
        cout << "destroyed" << endl;
    }
};
  • A destructor runs automatically when an object is destroyed.
  • Its name is the class name with a ~ in front.
  • It takes no arguments and has no return type.
  • There is only one destructor per class.

Its job: clean up anything the object set up.

The Whole Life

An Object's Lifecycle

The constructor runs at the start of an object's life; the destructor runs at the end, when it goes out of scope.

Worked Example 2

Watching Both Run

#include <iostream>
using namespace std;

class Demo {
public:
    Demo()  { cout << "created" << endl; }
    ~Demo() { cout << "destroyed" << endl; }
};

int main() {
    Demo d;                       // constructor runs
    cout << "using d" << endl;
    return 0;
}   // d goes out of scope -> destructor runs
Expected Output
created using d destroyed

What it does: "created" prints when d is made, and "destroyed" prints automatically when main ends and d goes out of scope.

The Real Use

Why Destructors Matter

Recall dynamic memory: every new needs a delete. If an object reserves memory with new, its destructor is the perfect place to free it.

Constructor

Reserves the resource — for example, new int[n].

Destructor

Frees the resource — the matching delete[].

The benefit: the cleanup happens automatically, so you cannot forget it.

Worked Example 3

A Class That Owns Memory

#include <iostream>
using namespace std;

class IntArray {
private:
    int *data;
    int size;
public:
    IntArray(int n) {          // constructor
        size = n;
        data = new int[n];     // reserve memory
    }
    ~IntArray() {              // destructor
        delete[] data;         // free memory
        cout << "freed" << endl;
    }
    void set(int i, int v) { data[i] = v; }
    int  get(int i) { return data[i]; }
};

int main() {
    IntArray a(3);
    a.set(0, 10);
    cout << a.get(0) << endl;   // 10
    return 0;
}   // destructor frees the memory here
Expected Output
10 freed

What it does: the constructor reserves an array with new[]; the destructor frees it with delete[] automatically when the object ends.

No leaks: you never have to remember the delete.

Be Careful

Common Mistakes

  • Giving a return type
    a constructor has none — not even void
  • Wrong name
    it must match the class name exactly
  • Forgetting the ~
    a destructor is ~ClassName()
  • Arguments on a destructor
    it always takes none
  • new without delete
    if the constructor uses new, the destructor must delete
  • Calling them by hand
    they run automatically; you do not call them

Remember: construct at birth, destruct at the end — both happen on their own.

Wrap Up

Summary

  • A constructor runs automatically when an object is created and sets up its data.
  • It has the class's name and no return type.
  • A default constructor takes no arguments; a class can have several constructors.
  • A destructor (~ClassName) runs automatically when the object is destroyed.
  • The destructor cleans up — for example, it frees memory the constructor reserved.
  • You never call them by hand; C++ runs them at the right time.

Constructors and destructors bookend an object's life — set up at the start, clean up at the end.
