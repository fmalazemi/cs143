The Core Distinction

Rotation Wraps · Shift Drops

Both move every value one place along. They differ only in what happens at the two ends.

Rotation

The value that falls off one end comes back at the other end.

1 2 3 4 5 rotate left → 2 3 4 5 1

Nothing is lost.

Shift

The value that falls off is lost. A 0 fills the empty place.

1 2 3 4 5 shift left → 2 3 4 5 0

One value is destroyed.

Keep this in mind: the same two words describe what happens to the bits of an integer.

Part A

Rotating and Shifting Arrays

Moving the values inside an array.

Part A · Rotate

Rotate Left by One

// save the first value, slide everyone left,
// then put the saved value at the end
int first = arr[0];

for (int i = 0; i < n - 1; i++) {
    arr[i] = arr[i + 1];    // pull the next one back
}

arr[n - 1] = first;         // wrap it around
Result
before: 1 2 3 4 5 after : 2 3 4 5 1
  • Save first, or it is overwritten immediately.
  • The loop stops at n - 2, because it reads arr[i + 1].
  • The saved value goes into the last slot.
Part A · Rotate

Rotate Right by One

// save the last value, slide everyone right,
// then put the saved value at the front
int last = arr[n - 1];

for (int i = n - 1; i > 0; i--) {
    arr[i] = arr[i - 1];    // push the previous one forward
}

arr[0] = last;              // wrap it around
Result
before: 1 2 3 4 5 after : 5 1 2 3 4

Notice the loop runs backwards. If it ran forwards, arr[i-1] would already be overwritten and every value would become arr[0].

Direction of the loop is the opposite of the direction of the move.

Worked Example 1

A Complete Rotation Program

#include <iostream>
using namespace std;

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int n = 5;

    // rotate left by one
    int first = arr[0];
    for (int i = 0; i < n - 1; i++)
        arr[i] = arr[i + 1];
    arr[n - 1] = first;

    for (int i = 0; i < n; i++)
        cout << arr[i] << " ";
    cout << endl;
    return 0;
}
Expected Output
2 3 4 5 1

What it does: every value moves one place left, and the old 1 reappears at the far right.

Change the loop and you get the right rotation instead.

Part A · Shift

Shift Left, Filling With Zero

// exactly the rotation, but we do NOT save
// the first value — we drop it
for (int i = 0; i < n - 1; i++) {
    arr[i] = arr[i + 1];
}

arr[n - 1] = 0;      // fill the empty place
Result
before: 1 2 3 4 5 after : 2 3 4 5 0

Compare with the rotation

Rotate leftShift left
Save first?yesno
Last slot getsfirst0

The loop is identical. Only the final line changes.

A shift is a rotation that forgets.

Part A · Shift

Shift Right, Filling With Zero

// run backwards, drop the last value
for (int i = n - 1; i > 0; i--) {
    arr[i] = arr[i - 1];
}

arr[0] = 0;          // fill the empty place
Result
before: 1 2 3 4 5 after : 0 1 2 3 4

OperationResult

Part A · By k Positions

Moving More Than One Place

The clearest way is a temporary array: compute where each value belongs, then copy back.

Rotate left by k

int tmp[5];
k = k % n;                       // in case k >= n

for (int i = 0; i < n; i++)
    tmp[i] = arr[(i + k) % n];   // wrap with %

for (int i = 0; i < n; i++)
    arr[i] = tmp[i];

Shift left by k (fill 0)

for (int i = 0; i < n; i++)
    tmp[i] = (i + k < n) ? arr[i + k] : 0;
With k = 2
start : 1 2 3 4 5 rotate left : 3 4 5 1 2 rotate right : 4 5 1 2 3 shift left : 3 4 5 0 0 shift right : 0 0 1 2 3

The % is the wrap. Rotation uses % n; a shift replaces the wrap with a 0.

Simpler alternative: repeat the by-one rotation k times.

Worked Example 2

Shift Left by k, Filling With Zero

#include <iostream>
using namespace std;

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int n = 5, k = 2;
    int tmp[5];

    for (int i = 0; i < n; i++) {
        if (i + k < n)
            tmp[i] = arr[i + k];   // pull from the right
        else
            tmp[i] = 0;            // past the end → 0
    }

    for (int i = 0; i < n; i++) arr[i] = tmp[i];

    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    return 0;
}
Expected Output
3 4 5 0 0

What it does: each slot i takes the value from i + k. When that index is past the end, we store 0.

Values 1 and 2 are gone — a shift destroys them.

Part B

Bitwise Operations

Every integer is a row of bits. Now we move those.

Part B · The Foundation

An Integer Is a Row of Bits

Inside the computer, an int is stored in binary — a row of 0s and 1s. Each position stands for a power of two.

place value : 128 64 32 16 8 4 2 1 5 : 0 0 0 0 0 1 0 1 3 : 0 0 0 0 0 0 1 1

Check it: for 5, the 4 and the 1 columns hold a 1, and 4 + 1 = 5.

Bitwise operators work on these columns, one column at a time.

Part B · The Operators

Six Bitwise Operators

OperatorNameA column gives 1 when…
a & bANDboth bits are 1
a | bORat least one bit is 1
a ^ bXORthe bits are different
~aNOTthe bit was 0 (it flips every bit)
a << kleft shift— moves bits left, 0 fills in
a >> kright shift— moves bits right, bits fall off

Careful: & is not &&, and | is not ||. The single symbols work on bits; the double symbols work on true/false.

Part B · Column by Column

AND, OR, XOR on 5 and 3

Worked Example 3

The Logic Operators in Code

#include <iostream>
using namespace std;

int main() {
    int x = 5;    // 0101
    int y = 3;    // 0011

    cout << (x & y) << endl;   // 1   both bits 1
    cout << (x | y) << endl;   // 7   either bit 1
    cout << (x ^ y) << endl;   // 6   bits differ
    cout << (~x)    << endl;   // -6  every bit flipped
    return 0;
}
Expected Output
1 7 6 -6

Why is ~5 equal to -6? Flipping every bit also flips the leftmost bit, which C++ uses to mark a negative number.

Always put bitwise expressions in brackets — << binds tighter than &.

Part B · The Shift Operators

Moving the Bits

The bits behave exactly like the values in a shifted array: they move along, the far ones fall off, and 0s fill the gaps.

a << k — left shift

5 << 1 → 10 0000 0101 → 0000 1010

Bits move left, 0 enters on the right.

a >> k — right shift

20 >> 2 → 5 0001 0100 → 0000 0101

Bits move right and fall off the end.

Same word, same meaning: the array shift and the bit shift both lose what falls off, and both fill with 0.

Part B · The Picture

What 5 << 1 Really Does

Part B · The Shortcut

Shifting Is Multiplying and Dividing by 2

cout << (5 << 1) << endl;    // 10  = 5 * 2
cout << (5 << 2) << endl;    // 20  = 5 * 4
cout << (5 << 3) << endl;    // 40  = 5 * 8

cout << (20 >> 1) << endl;   // 10  = 20 / 2
cout << (20 >> 2) << endl;   // 5   = 20 / 4

// a lost bit means a lost remainder
cout << (7 >> 1) << endl;    // 3   (7 / 2 = 3)
Expected Output
10 20 40 10 5 3
ShiftSame as
a << ka × 2ᵏ
a >> ka ÷ 2ᵏ

7 >> 1 is 3 — the bit that fell off was the remainder.

Worked Example 4

Useful Bit Tricks

int n = 12;      // 0000 1100

// is n even? test the lowest bit
if ((n & 1) == 0) cout << "even" << endl;

// read bit number 2 (counting from 0, right to left)
cout << ((n >> 2) & 1) << endl;    // 1

// turn ON bit 1
cout << (n | (1 << 1)) << endl;    // 14

// turn OFF bit 2
cout << (n & ~(1 << 2)) << endl;   // 8

// flip bit 0
cout << (n ^ (1 << 0)) << endl;    // 13
Expected Output
even 1 14 8 13

The pattern: 1 << k builds a mask — a number with a single 1 at position k.

  • | the mask to set a bit.
  • & ~ the mask to clear a bit.
  • ^ the mask to flip a bit.
Putting It Together

One Idea, Two Places

The array shift and the bit shift are the same operation on different data.

Array of intsBits of an int
Move leftloop: arr[i] = arr[i+1]a << 1
Move rightloop: arr[i] = arr[i-1]a >> 1
What falls offis lostis lost
What fills in00
Rotation exists?yes, you write itnot built in

The difference: C++ gives you << and >> for bits as a single fast instruction. For arrays, you write the loop yourself.

There is no built-in bit rotation operator — a rotation must be built from two shifts and an |.

Be Careful

Common Mistakes

Arrays

  • Not saving the end value
    rotation needs first or last stored before the loop
  • Looping the wrong way
    a right shift must run backwards, or all values become arr[0]
  • Reading past the end
    arr[i+1] needs i < n - 1

Bits

  • Writing && instead of &
    4 && 3 is 1, but 4 & 3 is 0
  • Forgetting brackets
    cout << x & y does not do what you expect
  • Shifting a negative number
    the result is not always what you predict

Rule: a shift always loses data; a rotation never does.

Wrap Up

Summary

  • A rotation wraps the end value around; a shift loses it and fills with 0.
  • Rotate left: save arr[0], slide left, put it at the end. Rotate right: the mirror, looping backwards.
  • To shift, use the same loop but store 0 in the empty slot.
  • Moving by k: use a temporary array; % n gives the wrap for a rotation.
  • &, |, ^, ~ work on one bit column at a time. They are not && and ||.
  • a << k multiplies by 2ᵏ; a >> k divides by 2ᵏ. Bits that fall off are lost, and 0s fill in.

A shift forgets. A rotation remembers.
