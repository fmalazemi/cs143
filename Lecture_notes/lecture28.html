The Problem

What Is Sorting?

Sorting means rearranging the items of an array so they are in order — usually smallest to largest.

Before

5 1 4 2

After

1 2 4 5

All three algorithms today use the same two tools you already know: comparing two values, and swapping them.

// swapping two values in an array
int temp = arr[i];
arr[i] = arr[j];
arr[j] = temp;
Part A

Bubble Sort

Compare each pair of neighbours, and swap them if they are out of order.

Part A · The Idea

Swap Neighbours, Again and Again

The rule

  • Walk along the array from left to right.
  • Compare each item with the one next to it.
  • If they are in the wrong order, swap them.
  • Repeat until no swaps are needed.

Why "bubble"?

After each pass, the largest remaining value has moved — bubbled — all the way to the right end, where it stays.

Key point: after pass 1 the last item is correct, after pass 2 the last two are correct, and so on.

Part A · Trace

One Pass of Bubble Sort

Starting from 5 1 4 2, comparing each neighbouring pair.

Worked Example 1

Bubble Sort in C++

#include <iostream>
using namespace std;

int main() {
    int arr[4] = {5, 1, 4, 2};
    int n = 4;

    // each pass fixes one more value at the end
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {   // out of order?
                int temp = arr[j];       // swap them
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }

    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    return 0;
}
Expected Output
1 2 4 5

What it does: the inner loop compares neighbours and swaps. The - i skips the values already parked at the end.

Two nested loops: about n × n comparisons.

Part B

Selection Sort

Find the smallest remaining value, and move it into place.

Part B · The Idea

Find the Smallest, Put It First

The rule

  • Look through the unsorted part and find the smallest value.
  • Swap it with the first unsorted position.
  • That position is now finished; move one step right.
  • Repeat until the whole array is sorted.

The difference

Bubble sort swaps many times per pass. Selection sort searches first, then makes one swap per pass.

Key point: the array grows sorted from the left.

Part B · Trace

Selection Sort Step by Step

Worked Example 2

Selection Sort in C++

#include <iostream>
using namespace std;

int main() {
    int arr[4] = {5, 1, 4, 2};
    int n = 4;

    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;              // assume i is smallest

        // search the rest for something smaller
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }

        if (minIndex != i) {           // one swap per pass
            int temp = arr[i];
            arr[i] = arr[minIndex];
            arr[minIndex] = temp;
        }
    }

    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    return 0;
}
Expected Output
1 2 4 5

What it does: the inner loop finds the smallest value's position. The swap happens once, after the search.

Fewer swaps than bubble sort, but the same number of comparisons.

Part C

Insertion Sort

Take the next value and slide it back into its correct place.

Part C · The Idea

Like Sorting Cards in Your Hand

The rule

  • Treat the left part of the array as already sorted.
  • Take the next value — the key.
  • Slide larger values one step right to make a gap.
  • Drop the key into the gap.

The picture

Exactly how most people sort playing cards: pick up the next card and push it back until it sits in the right spot.

Key point: insertion sort shifts values rather than swapping pairs.

Part C · Trace

Inserting Each Value

Worked Example 3

Insertion Sort in C++

#include <iostream>
using namespace std;

int main() {
    int arr[4] = {5, 1, 4, 2};
    int n = 4;

    for (int i = 1; i < n; i++) {
        int key = arr[i];       // the value to place
        int j = i - 1;

        // slide bigger values one step right
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;       // drop it in the gap
    }

    for (int i = 0; i < n; i++) cout << arr[i] << " ";
    return 0;
}
Expected Output
1 2 4 5

What it does: the while loop shifts larger values right until it finds the gap, then places key.

Starts at i = 1: one item alone is already sorted.

Putting It Together

Comparing the Three

AlgorithmCore ideaSwapsBest caseWorst case
Bubbleswap neighboursmanynn × n
Selectionfind the smallestone per passn × nn × n
Insertioninsert into sorted partshiftsnn × n
  • All three are slow for large arrays — roughly n × n work.
  • Insertion sort is the fastest of the three on data that is already nearly sorted.
  • Selection sort makes the fewest swaps — useful when swapping is expensive.

For real work, use the STL: sort(v.begin(), v.end());

Be Careful

Common Mistakes

  • Reading past the end
    bubble sort compares arr[j+1], so j must stop at n-2
  • Losing a value in a swap
    always use a temp variable
  • Starting insertion sort at i = 0
    begin at i = 1; the first item is already sorted
  • Swapping inside the search
    selection sort swaps after finding the minimum
  • Forgetting j >= 0
    the insertion while must not run past the left end
  • Tracking the value, not the index
    selection sort stores minIndex, not the minimum value

Advice: trace each algorithm on paper with four numbers before you trust your code.

Wrap Up

Summary

  • Sorting rearranges an array into order, using only comparisons and swaps.
  • Bubble sort: swap neighbours; the largest value bubbles to the right each pass.
  • Selection sort: find the smallest remaining value; one swap per pass.
  • Insertion sort: slide each new value back into the sorted left part.
  • All three use nested loops and take about n × n steps.
  • Insertion sort wins on nearly-sorted data; selection sort makes the fewest swaps.

Learn these three by hand — they teach how loops, comparisons, and swaps work together.
