Why Do We Need Loops?

‚ùå Without loops

cout << "Hello!\n";
cout << "Hello!\n";
cout << "Hello!\n";
// ... 997 more times?!

Tedious, error-prone, unmaintainable.

‚úÖ With a loop

for (int i = 0; i < 1000; i++) {
    cout << "Hello!\n";
}

Three lines. Any count. Easy to change.

Loops let us express repetition once and let the computer do the repeating. This is the heart of what makes programming powerful.

The while Loop

Syntax

while (condition) {
    // body ‚Äî repeats while true
}

When to use

Use while when you don't know in advance how many iterations you need ‚Äî the number depends on runtime input or a changing state.

Execution Flow

1 Evaluate condition
‚úó If false ‚Üí skip to after the loop
‚úì If true ‚Üí run the body
4 Go back to step 1

‚ö†Ô∏è If the condition is never false, you get an infinite loop.

while Example: Count 1 to N

int i = 1;
int N = 5;

while (i <= N) {
    cout << i << " ";
    i++;

Common while Mistakes

‚ùå Infinite Loop ‚Äî Forgetting to Update

int i = 1;
while (i <= 5) {
    cout << i;   // prints 1 forever!
    // BUG: forgot i++;
}

‚úÖ Always make sure the loop variable changes so the condition eventually becomes false.

‚ùå Off-by-One Error

int i = 0;
while (i <= 5) { // runs 6 times (0..5), not 5
    cout << i << " ";
    i++;
}

‚úÖ Double-check your start value and whether < vs <= gives you the right count.

‚ùå Wrong Condition Direction

int i = 10;
while (i > 0) {
    cout << i;
    i++;  // BUG: should be i--
}

‚úÖ The update must move the variable toward the termination condition, not away from it.

The for Loop

Syntax

for (init; condition; update) {
    // body
}
init runs once before the loop
condition checked before each iteration
update runs after each iteration

When to use

Use for when you know the range ‚Äî counting from A to B, iterating over an array, or repeating N times.

Execution Order

1 init ‚Äî execute once
2 Evaluate condition
‚úó If false ‚Üí exit loop
‚úì If true ‚Üí run body
5 Run update
6 Go back to step 2

Python equivalent:
for i in range(0, 10): ‚âà for(int i=0; i<10; i++)

for Example: Sum of 1 to N

int N = 5;
int sum = 0;

for (int i = 1; i <= N; i++) {
    sum += i;

cout << "Sum = " << sum;

The do-while Loop

Key Idea

The body executes at least once ‚Äî the condition is checked after each iteration.

Syntax

do {
    // body ‚Äî runs first
} while (condition);  // note the semicolon!

Example: Input Validation Menu

int choice;
do {
    cout << "Menu:\n";
    cout << "1. Play\n";
    cout << "2. Settings\n";
    cout << "3. Quit\n";
    cout << "Enter choice: ";
    cin  >> choice;
} while (choice < 1 || choice > 3);

cout << "You chose: " << choice;

The menu is always shown at least once. If the user enters an invalid number, it repeats.

break and continue

break ‚Äî Exit the loop immediately

for (int i = 1; i <= 100; i++) {
    if (i == 5) break;
    cout << i << " ";
}
// Output: 1 2 3 4

When i reaches 5, the loop stops entirely.

continue ‚Äî Skip to next iteration

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    cout << i << " ";
}
// Output: 1 2 4 5

When i is 3, the rest of the body is skipped, but the loop keeps going.

‚ö†Ô∏è Use break and continue sparingly. Over-use makes code harder to follow. Prefer restructuring the condition when possible.

Nested Loops

Printing a Grid Pattern

for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 4; col++) {
        cout << "* ";
    }
    cout << "\n";
}

How It Works

The outer loop controls the rows.

For each row, the inner loop runs completely to print all columns.

Total iterations = rows √ó columns = 3 √ó 4 = 12

Summary ‚Äî Key Takeaways

    

      

        
Loops let you repeat code without rewriting it. They are one of the most fundamental tools in programming.

      

      

        
for ‚Äî use when you know the count. ¬∑ while ‚Äî use when it depends on a condition. ¬∑ do-while ‚Äî use when the body must run at least once.

      

      

        
Every loop needs a condition that eventually becomes false. If it doesn't, you have an infinite loop.

      

      

        
break exits a loop early. continue skips to the next iteration. Use both with care.

      

      

        
Nested loops let you work in two (or more) dimensions ‚Äî grids, tables, pattern generation.
