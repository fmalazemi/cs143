Part A

The Partition Problem

Forget sorting. Just put one value where it belongs.

Part A · The Problem

Where Does arr[0] Belong?

Take the first value of the array. Call it the pivot. In the final sorted array, it will sit at some index p. Find that index — and move the pivot there.

Given

arr = 5 3 8 1 9 2 7 pivot = arr[0] = 5

Wanted

2 3 1 [5] 9 8 7 ↑ index 3 — correct!

Three values (3, 1, 2) are smaller than 5, so 5 must end up at index 3. Everything smaller goes to its left; everything larger goes to its right.

Part A · The Goal

Smaller · Pivot · Larger

Part A · The Key Point

One Value Is Finished Forever

What partition gives you

  • The pivot is at its correct, final index.
  • Everything left of it is smaller.
  • Everything right of it is larger.

What it does not give you

  • A sorted left side.
  • A sorted right side.
2 3 1 [5] 9 8 7 ^^^^^ ^^^^^ messy messy

This is the whole trick. Partition solves the problem for one value with certainty — and it never has to be reconsidered.

Part A · The Method

Sweep With j, Grow With i

Two indices

  • j — walks through every value, one by one.
  • i — marks the end of the "smaller" region.

The rule

When arr[j] < pivot:

  • Grow the smaller region: i++
  • Swap arr[i] with arr[j]

Otherwise, do nothing — just let j move on.

At the very end, swap the pivot at arr[left] with arr[i]. The pivot drops into place, and i is the answer.

Part A · A Snapshot

The Four Regions

Halfway through 5 3 8 1 9 2 7, the array has become 5 3 1 8 9 2 7, with i = 2 and j = 5.

Part A · The Code

The partition Function

// puts arr[left] in its correct place,
// and returns that place
int partition(int arr[], int left, int right) {
    int pivot = arr[left];    // the value we place
    int i = left;             // end of "smaller" region

    for (int j = left + 1; j <= right; j++) {
        if (arr[j] < pivot) {
            i++;                       // grow the region
            int temp = arr[i];         // swap arr[i], arr[j]
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }

    // finally, drop the pivot into place
    int temp = arr[left];
    arr[left] = arr[i];
    arr[i] = temp;

    return i;                 // the pivot's correct index
}
  • j starts at left + 1 — never compare the pivot with itself.
  • i starts at left, so the first smaller value lands at left + 1.
  • The final swap is outside the loop.

Cost: the loop touches each value once, so partition takes about n steps.

Part B

From Partition to Quick Sort

One value is correct. Now repeat on what is left.

Part B · The Insight

Repeat on Each Side

After partition, the pivot is finished. The two sides are still messy — but each is just a smaller version of the same problem.

2 3 1 [5] 9 8 7 └───┬───┘ └───┬───┘ partition it partition it ↓ ↓ 1 [2] 3 [5] [7] 8 9 ...and so on

Each partition fixes one more value forever. When every piece has shrunk to a single value, there is nothing left to fix — the array is sorted.

That is recursion again: partition, then quick sort the left, then quick sort the right.

Part B · The Code

The quickSort Function

void quickSort(int arr[], int left, int right) {
    // base case: 0 or 1 value needs no work
    if (left >= right) return;

    // place one value correctly
    int p = partition(arr, left, right);

    quickSort(arr, left, p - 1);    // sort the smaller side
    quickSort(arr, p + 1, right);   // sort the larger side
}
  • p is the pivot's final index.
  • We recurse on left..p-1 and p+1..right.
  • Index p is skipped — that value is already correct.

Compare with merge sort

Merge sort does its work after the recursive calls. Quick sort does its work before them.

Call it with quickSort(arr, 0, n - 1);

Part B · The Picture

Every Partition Locks One Value

Worked Example 2

The Complete Program

#include <iostream>
using namespace std;

int partition(int arr[], int left, int right) {
    int pivot = arr[left];
    int i = left;
    for (int j = left + 1; j <= right; j++) {
        if (arr[j] < pivot) {
            i++;
            int t = arr[i]; arr[i] = arr[j]; arr[j] = t;
        }
    }
    int t = arr[left]; arr[left] = arr[i]; arr[i] = t;
    return i;
}

void quickSort(int arr[], int left, int right) {
    if (left >= right) return;
    int p = partition(arr, left, right);
    quickSort(arr, left, p - 1);
    quickSort(arr, p + 1, right);
}

int main() {
    int arr[7] = {5, 3, 8, 1, 9, 2, 7};
    quickSort(arr, 0, 6);
    for (int i = 0; i < 7; i++) cout << arr[i] << " ";
    return 0;
}
Expected Output
1 2 3 5 7 8 9

What it does: each call to partition fixes one value; the recursion handles what remains on either side.

No extra array is needed — quick sort rearranges the values in place.

Tracing the Calls

Partitions in order
quickSort [5 3 8 1 9 2 7] pivot=5 → [2 3 1] 5 [9 8 7] 5 fixed at 3 quickSort [2 3 1] pivot=2 → [1] 2 [3] 2 fixed at 1 quickSort [9 8 7] pivot=9 → [7 8] 9 [] 9 fixed at 6 quickSort [7 8] pivot=7 → [] 7 [8] 7 fixed at 4 final: 1 2 3 5 7 8 9

What to notice

  • Each line locks exactly one value into place.
  • Pieces of size 1 return immediately — the base case.
  • A partition can be uneven: [9 8 7] gave [7 8] and an empty side.

Four partitions fixed 4 values; the other 3 were finished as single-value pieces.

Part B · The Catch

The Worst Case: An Already Sorted Array

If arr[0] is always the smallest value, then nothing goes to its left. Each partition removes only one value.

Sorted input

1 2 3 4 5 6 7 8 pivot 1 → left empty pivot 2 → left empty ... n times

The tree becomes one long chain of depth n.

Balanced input

4 2 6 1 3 5 8 each pivot lands near the middle

The tree has depth log₂(n).

n = 8, already sorted : 28 comparisons ( = n(n-1)/2, the worst possible ) n = 7, balanced : 11 comparisons

A common fix: choose the pivot from the middle, or at random.

Putting It Together

Quick Sort and Merge Sort

Quick sortMerge sort
Average casen × log₂(n)n × log₂(n)
Worst casen × nn × log₂(n)
Extra memorynone (in place)a second array
Work happensbefore the callsafter the calls
Splittingby value (the pivot)by position (the middle)

The trade: quick sort uses no extra memory and is usually faster in practice, but a bad pivot ruins it. Merge sort is slower on average but never has a bad day.

Be Careful

Common Mistakes

  • Starting j at left
    the pivot would be compared with itself
  • Forgetting the final swap
    the pivot never reaches its place
  • Including the pivot in a recursive call
    use p-1 and p+1, never p
  • No base case
    if (left >= right) return; must come first
  • Expecting sorted halves
    partition only separates; it does not sort
  • Returning the wrong index
    return i after the swap, not j

The infinite-recursion test: including index p in a recursive call means the piece never shrinks.

2 · Why skip p?

Because the value at p is already in its final, correct position. Including it would also stop the pieces from shrinking, causing infinite recursion.

3 · Is the left side sorted?

No. Partition only guarantees that every value on the left is smaller than the pivot — their order among themselves is untouched.

Solutions (4–6)

Exercise Answers

4 · The base case

if (left >= right) return;

A piece holding one value — or none — is already sorted, so there is nothing to do.

6 · Why no second array?

Partition rearranges values by swapping inside the array. Merge must read two sorted halves while writing the result, so it needs copies to avoid overwriting.

5 · Why sorted input is worst

arr[0] is then the smallest value, so nothing is smaller than it and the left side is always empty.

each partition removes just ONE value depth = n, not log(n) n=8 sorted → 28 comparisons n=7 balanced → 11
Wrap Up

Summary

  • The partition problem: find the correct final index of the pivot arr[0], and move it there.
  • j sweeps the array; i marks the end of the smaller region; swap when arr[j] < pivot.
  • A final swap drops the pivot at i — and it never moves again.
  • Partition separates but does not sort the two sides.
  • Quick sort = partition, then quick sort the left side and the right side, skipping the pivot.
  • Average n × log₂(n), in place. Worst case n × n on already sorted input.

Place one value with certainty — then let recursion do it again and again.
