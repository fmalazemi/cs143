02 · From Python

Static Types vs. Dynamic Types

In Python a variable can hold anything and can change type at any time. In C++ you choose a type when you declare a variable, and it keeps that type.

Python (dynamic)

x = 5        # an int now
x = "hello"  # a string now — allowed

C++ (static)

int x = 5;       // x is an int, always
// x = "hello";  // ERROR: wrong type

Why it matters: a fixed type fixes the size and range of a variable — which leads to today's topics.

03 · Part A · Variable Types

Choosing and Declaring a Type

#include <iostream>
#include <string>
using namespace std;

int main() {
    int age = 20;          // whole number
    double price = 4.50;   // decimal number
    char grade = 'A';      // one character
    bool passed = true;    // true / false
    string name = "Sara";  // text

    cout << name << " " << age << " " << grade << endl;
    return 0;
}
Expected Output
Sara 20 A
  • Pick the smallest type that fits your data.
  • Use int for counting, double for money or measurements.
  • Use char for one letter, string for words.
04 · Part B · Scope

Local and Block Scope

Scope is the region of code where a variable can be used. A variable is visible inside the braces { } where it was declared.

04 · Part B · Scope

Global Variables

#include <iostream>
using namespace std;

int counter = 0;   // GLOBAL: declared outside all functions

void addOne() {
    counter++;      // can use the global directly
}

int main() {
    addOne();
    addOne();
    cout << counter << endl;   // 2
    return 0;
}
Expected Output
2
  • A global variable lives outside every function.
  • Every function can read and change it.
  • It exists for the whole run of the program.

Caution: use globals sparingly — when many functions change one, bugs are hard to find.

Worked Example 1

Global vs. Local (Shadowing)

#include <iostream>
using namespace std;

int x = 100;        // global x

void show() {
    int x = 5;      // local x hides the global
    cout << "Inside: " << x << endl;   // 5
}

int main() {
    show();
    cout << "Global: " << x << endl;    // 100
    return 0;
}
Expected Output
Inside: 5 Global: 100

Shadowing: when a local variable has the same name as a global, the local one is used inside that function. The global is untouched.

Advice: avoid reusing names — it confuses readers.

05 · Part C · Overflow & Underflow

What Is Overflow?

Each type can hold only a limited range. Overflow happens when a value goes past the maximum — it wraps around to the minimum, like a car odometer rolling over.

Result: the number becomes wrong (often negative). The fix is to use a bigger type, such as long long.

Worked Example 2

Overflow in Action

#include <iostream>
using namespace std;

int main() {
    int big = 2147483647;       // the largest int
    cout << big + 1 << endl;     // overflow!

    long long safe = 2147483647;
    cout << safe + 1 << endl;    // plenty of room
    return 0;
}
Expected Output
-2147483648 2147483648

What happens: adding 1 to the largest int wraps to the most negative number. The long long version has room, so it is correct.

Lesson: choose a type big enough for the values you expect.

05 · Part C · Overflow & Underflow

Underflow

Unsigned underflow (wrap down)

unsigned int u = 0;
cout << u - 1 << endl;
// 4294967295  (wraps to the top)

An unsigned type cannot be negative, so going below 0 wraps to its largest value.

Float underflow (too small)

double tiny = 1e-310;
tiny = tiny / 1e100;
cout << tiny << endl;   // 0

A decimal number can be so small that it cannot be stored, so it becomes 0.

In short: overflow is "too big to fit"; underflow is "too small to fit" (below the minimum, or too close to zero).

06 · Part D · Casting

Implicit Conversion & Truncation

#include <iostream>
using namespace std;

int main() {
    int a = 5;
    double d = a;       // int -> double (automatic)
    cout << d << endl;   // 5

    double pi = 3.9;
    int n = pi;         // double -> int: decimals DROPPED
    cout << n << endl;   // 3  (not 4!)
    return 0;
}
Expected Output
5 3
  • Widening (int → double) is safe and automatic.
  • Narrowing (double → int) truncates: it cuts off the decimal part, it does not round.
  • So 3.9 becomes 3.
06 · Part D · Casting

Explicit Casting

Two ways to cast

double pi = 3.9;

int a = (int)pi;             // C-style cast
int b = static_cast<int>(pi); // C++ style

cout << a << " " << b;       // 3 3

Casting tells C++ to change a value from one type to another on purpose.

  • (int)x is the short C-style form.
  • static_cast<int>(x) is the preferred C++ form — clearer and safer.
  • Both drop the decimal part when going to int.
Worked Example 3

The Integer-Division Trap

#include <iostream>
using namespace std;

int main() {
    int a = 7, b = 2;

    cout << a / b << endl;                  // 3  (wrong!)
    cout << (double)a / b << endl;          // 3.5
    cout << static_cast<double>(a) / b;     // 3.5
    return 0;
}
Expected Output
3 3.5 3.5

Why: when both sides are int, C++ does integer division and throws away the fraction.

Fix: cast one value to double first, so the division keeps the decimals.

07 · Be Careful

Common Mistakes

  • Integer division
    7 / 2 gives 3; cast to double for 3.5
  • Assuming int is unlimited
    large sums overflow; use long long
  • Expecting rounding
    double → int truncates, it does not round
  • Using a variable out of scope
    a variable does not exist outside its braces
  • Name shadowing
    a local name can hide a global one by accident
  • Mixing signed and unsigned
    0u - 1 becomes a huge number

11 · Wrap Up

Summary

  • C++ is statically typed: each variable has a fixed type with a fixed size and range.
  • Main types: int, long long, double, char, bool, string.
  • Scope is where a variable lives; local stays in its braces, global is everywhere.
  • Overflow = past the maximum; underflow = below the minimum or too tiny to store.
  • Casting converts types; narrowing to int truncates the decimals.
  • Cast to double before dividing to keep the fraction; use long long for big values.

These habits prevent some of the most common beginner bugs in C++.
