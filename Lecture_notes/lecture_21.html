Today's Goals

Learning Objectives

By the end of this lecture, you will be able to:

  • Use the -> operator to reach a member through a pointer.
  • Show that ptr->x is the same as (*ptr).x.
  • Create an object on the heap with new.

The Setup

Two Ideas We Already Have

A member with the dot

struct Point {
    int x;
    int y;
};

Point p;
p.x = 3;      // dot reads a member

The dot . reaches a member of an object we hold directly.

A pointer holds an address

Point p;
Point* ptr = &p;   // ptr points to p

A pointer does not hold the object — it holds its address.

The question: when we only have a pointer, how do we reach the object's members?

Part A

The Arrow Operator  ->

Reaching a member through a pointer.

Part A · The Problem

The Dot Does Not Work on a Pointer

Point p;
p.x = 3;

Point* ptr = &p;

ptr.x = 5;        // WRONG: ptr is a pointer, not a Point

(*ptr).x = 5;     // correct, but clumsy to write
  • ptr is an address, so ptr.x is not allowed.
  • We must first dereference with *ptr to get the object, then use the dot.
  • The parentheses in (*ptr).x are required — and easy to forget.

C++ gives us a cleaner tool for exactly this.

Part A · The Solution

The Arrow Operator  ->

The -> operator follows a pointer to its object, then reads a member — in one clean step.

Part A · The Syntax

How to Write It

pointer->member
  • pointer — a pointer to an object (for example Point*).
  • -> — a dash - followed by a greater-than sign >, with no space between them.
  • member — the field or method you want to reach.

It works for both data members (ptr->x) and methods (ptr->area()).

Part A · The Key Fact

-> Is a Shorthand

The arrow is exactly the same as dereferencing and then using the dot. Both reach the same member.

Worked Example 1

A Struct Through a Pointer

#include <iostream>
using namespace std;

struct Point {
    int x;
    int y;
};

int main() {
    Point p;
    p.x = 3;
    p.y = 4;

    Point* ptr = &p;         // ptr points to p

    // reach members through the pointer
    cout << ptr->x << endl;   // 3
    cout << ptr->y << endl;   // 4
    return 0;
}
Expected Output
3 4

What it does: ptr holds the address of p. ptr->x follows that address and reads the member x.

Same as: (*ptr).x, only clearer.

Worked Example 2

Calling Methods Through a Pointer

#include <iostream>
using namespace std;

class Counter {
private:
    int count;
public:
    Counter() : count(0) { }
    void increment() { count++; }
    int get() { return count; }
};

int main() {
    Counter c;
    Counter* ptr = &c;       // pointer to the object

    ptr->increment();        // call a method
    ptr->increment();
    cout << ptr->get() << endl;   // 2
    return 0;
}
Expected Output
2

What it does: the arrow works for methods too. ptr->increment() calls increment on the object that ptr points to.

this-> uses this same arrow: this is a pointer to the current object.

Part A · A Real Use

Following a Chain of Pointers

#include <iostream>
using namespace std;

struct Node {
    int value;
    Node* next;      // a pointer to another Node
};

int main() {
    Node a = {1, nullptr};
    Node b = {2, nullptr};
    a.next = &b;             // link a to b

    Node* p = &a;
    cout << p->value << endl;        // 1
    cout << p->next->value << endl;  // 2
    return 0;
}
Expected Output
1 2

What it does: p->next gives the pointer to the next node, and a second ->value reads its value.

Read it left to right: follow next, then take value.

Part B

Creating Objects With new / delete

Making objects that live on the heap, and freeing them by hand.

Part B · Motivation

Why Create Objects With new?

A normal object (on the stack)

void f() {
    Box b;      // created here
}               // destroyed here, automatically

It disappears at the end of its block. You cannot keep it afterwards.

A dynamic object (on the heap)

Box* b = new Box;   // created here
// ... lives on ...
delete b;           // destroyed only when YOU say so

It lives until you delete it — useful when an object must outlive the function that made it.

Key point: new gives back a pointer — which is why we reach its members with ->.

Part B · Creating

new — One Object on the Heap

Part B · Constructor Arguments

Passing Values to new

class Box {
    int width, height;
public:
    // constructor with two values
    Box(int w, int h) : width(w), height(h) { }
    int area() { return width * height; }
};

int main() {
    // new runs the constructor with 5 and 3
    Box* b = new Box(5, 3);

    cout << b->area() << endl;   // 15

    delete b;                    // free it
    return 0;
}
Expected Output
15
  • new Box(5, 3) reserves memory and runs the constructor.
  • The result is a pointer, so we use b->area().
  • delete b gives the memory back.
Worked Example 3

new Builds, delete Cleans Up

#include <iostream>
using namespace std;

class Box {
    int w, h;
public:
    Box(int w, int h) : w(w), h(h) {
        cout << "Box made" << endl;   // constructor
    }
    ~Box() {
        cout << "Box gone" << endl;   // destructor
    }
    int area() { return w * h; }
};

int main() {
    Box* b = new Box(5, 3);      // "Box made"
    cout << b->area() << endl;    // 15
    delete b;                    // "Box gone"
    return 0;
}
Expected Output
Box made 15 Box gone

What it does: new runs the constructor; delete runs the destructor and frees the memory. Every new needs exactly one delete.

Part B · Many Objects

Arrays of Objects: new[] and delete[]

// make 3 objects on the heap
Box* arr = new Box[3];

// arr is a POINTER, but arr[0] is an OBJECT
arr[0].setSize(2, 3);   // use the dot on arr[0]
arr[1].setSize(4, 5);

// free the WHOLE array
delete[] arr;           // note the [ ]
  • new Box[3] makes three objects; each is built with the default constructor.
  • arr is a pointer, but arr[0] is an object — so use the dot on it.
  • Match new[] with delete[], never plain delete.
Worked Example 4

An Array of Objects

#include <iostream>
using namespace std;

struct Point {
    int x;
    int y;
};

int main() {
    Point* pts = new Point[2];   // 2 Points on the heap

    pts[0].x = 1;  pts[0].y = 2; // pts[0] is an object
    pts[1].x = 3;  pts[1].y = 4;

    cout << pts[0].x << " " << pts[1].y << endl;  // 1 4

    delete[] pts;                // free the whole array
    return 0;
}
Expected Output
1 4

What it does: pts points to the first of two heap objects. Each pts[i] is an object, so we use the dot. delete[] frees all of them at once.

A Short Comparison

How Python Differs

PythonC++
Through a pointer— (no pointers)ptr->x
Create an objectb = Box()Box* b = new Box;
Free itautomaticdelete b;

Python creates objects with Box() and frees them for you automatically. C++ gives you direct control with new and delete — and with that control comes the duty to free what you create.

More power, more responsibility: a forgotten delete leaks memory.

Be Careful

Common Mistakes

Dot and arrow

  • Dot on a pointer
    ptr.x — use ptr->x
  • Arrow on an object
    p->x when p is a Point — use p.x
  • Missing parentheses
    *ptr.x means *(ptr.x) — write (*ptr).x

new and delete

  • Forgetting delete
    a new object never freed leaks memory
  • Wrong delete form
    use delete[] for new[], not plain delete
  • Using after delete
    the pointer is now dangling — do not touch it

Rule of thumb: pointer on the left → ->; object on the left → .; every new → one delete.

Wrap Up

Summary

  • The -> operator reaches a member through a pointer; ptr->x equals (*ptr).x.
  • new creates an object on the heap and returns a pointer to it.
  • You can pass constructor arguments: new Box(5, 3).
  • delete runs the destructor and frees one object; delete[] frees an array.
  • Objects from new are used with ->; every new needs one matching delete.
  • Rule: pointer → ->; object → .

Pointers, the arrow, and new / delete work together to manage objects by hand.
