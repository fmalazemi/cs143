Motivation

Numbers and Text Are Different Things

cout hides the difference, so it is easy to forget that 42 and "42" are not the same.

A number

int n = 42;

You can do arithmetic: n + 1 is 43.

A string of text

string s = "42";

You can find its length, or read s[0]. Arithmetic makes no sense.

We often need to move between them: reading input, building messages, or working on digits one at a time.

The Map

Three Types, Six Conversions

Part A

int ⇄ string ⇄ char

Moving between numbers and text.

Part A · Numbers to Text

int → string

#include <string>
using namespace std;

int n = 42;
string s = to_string(n);   // "42"

cout << s.length() << endl;   // 2
cout << s + "!" << endl;      // 42!

The older way

#include <sstream>
stringstream ss;
ss << 42;
string s = ss.str();       // "42"
Expected Output
2 42!
  • to_string works for int, double, long, and more.
  • Needs #include <string>.
  • Once it is a string, you can join it with +.

Use to_string. The stringstream way is older and longer.

Part A · Text to Numbers

string → int

#include <string>
using namespace std;

string s = "123";

int n = stoi(s);            // string to int
cout << n + 1 << endl;      // 124

double d = stod("3.14");    // string to double
cout << d << endl;          // 3.14
Expected Output
124 3.14

The family

FunctionGives
stoiint
stoddouble
stollong

stoi("hello") is an error — the text must look like a number.

Worked Example 1

A Round Trip

#include <iostream>
#include <string>
using namespace std;

int main() {
    int age = 20;

    // number -> text, to build a message
    string msg = "Age: " + to_string(age);
    cout << msg << endl;

    // text -> number, to do arithmetic
    string input = "15";
    int years = stoi(input);
    cout << "Next year: " << years + 1 << endl;

    return 0;
}
Expected Output
Age: 20 Next year: 16

What it does: to_string lets us join a number onto text with +. stoi lets us do arithmetic on text.

Note: "Age: " + age without to_string is an error.

Part A · The Key Idea

A char Is Really a Small Number

Every character is stored as a number — its ASCII code. The characters '0' to '9' sit next to each other, in order.

Part A · The Classic Trap

char c = 5; Is Not the Character '5'

Wrong

char c = 5;
cout << c;   // prints nothing visible!

ASCII code 5 is an invisible control character, not the digit five.

Correct

char c = '0' + 5;
cout << c;   // prints 5

ASCII code 53 is the character '5'.

char c = 5; ascii code 5 → prints nothing char c = '0'+5; ascii code 53 → prints 5

Rule: to turn a digit d into a character, add '0'.

Part A · Both Directions

Digit ⇄ Character, and ASCII Codes

char d = '7';

// character -> the digit it shows
cout << d - '0' << endl;    // 7

// character -> its ASCII code
cout << (int)d << endl;     // 55
cout << (int)'A' << endl;   // 65

// a number -> the character with that code
cout << (char)66 << endl;   // B

// careful: this is arithmetic, not text
cout << '5' + 1 << endl;         // 54
cout << (char)('5' + 1) << endl; // 6
Expected Output
7 55 65 B 54 6

Notice: '5' + 1 gives 54, a number. Cast it back to char to see '6'.

Use - '0' for the digit value; use (int) for the ASCII code.

Worked Example 2

Sum the Digits of a String

#include <iostream>
#include <string>
using namespace std;

int main() {
    string s = "4729";
    int sum = 0;

    for (int i = 0; i < s.length(); i++) {
        // s[i] is a char; subtract '0' to get its value
        sum += s[i] - '0';
    }

    cout << "Sum of digits: " << sum << endl;
    return 0;
}
Expected Output
Sum of digits: 22

What it does: 4 + 7 + 2 + 9 = 22. Each s[i] is a character, so - '0' converts it to its digit value.

Without - '0' you would add ASCII codes and get 214.

Worked Example 3

Building a String Digit by Digit

#include <iostream>
#include <string>
using namespace std;

int main() {
    int n = 407;
    string out = "";

    if (n == 0) out = "0";

    while (n > 0) {
        int digit = n % 10;              // last digit
        out = char('0' + digit) + out;   // put it in front
        n = n / 10;                      // drop it
    }

    cout << out << endl;
    return 0;
}
Expected Output
407

What it does: this is to_string written by hand. % 10 takes the last digit, / 10 removes it.

Why put it in front? Digits come out backwards, so each new one goes before the rest.

In real code, just use to_string(407).

Nearly all of these take and return a double.

Part B · The Two Most Used

sqrt and pow

#include <iostream>
#include <cmath>
using namespace std;

int main() {
    cout << sqrt(16) << endl;     // 4
    cout << sqrt(2) << endl;      // 1.41421

    cout << pow(2, 10) << endl;   // 1024
    cout << pow(5, 3) << endl;    // 125

    // a fractional power is a root
    cout << pow(9, 0.5) << endl;  // 3
    return 0;
}
Expected Output
4 1.41421 1024 125 3
  • There is no ** operator in C++ — always use pow.
  • cout shows about 6 digits by default, so sqrt(2) appears as 1.41421.
Part B · Rounding

floor, ceil, and round

Remember: floor always goes left on the number line, ceil always goes right — even for negatives.

Part B · Remainders

% for Integers, fmod for Decimals

#include <cmath>

cout << 10 % 3 << endl;        // 1   (int only)
cout << fmod(10, 3) << endl;   // 1
cout << fmod(10.5, 3) << endl; // 1.5

// integer division surprises beginners
cout << 10 / 3 << endl;        // 3
cout << 10.0 / 3 << endl;      // 3.33333
Expected Output
1 1 1.5 3 3.33333
  • % refuses to work on double.
  • 10 / 3 is 3: two ints give an int.
  • Make one side a double to keep the fraction.
Worked Example 4

Distance Between Two Points

#include <iostream>
#include <cmath>
using namespace std;

int main() {
    double x1 = 0, y1 = 0;
    double x2 = 3, y2 = 4;

    double dx = x2 - x1;          // 3
    double dy = y2 - y1;          // 4

    // distance = square root of (dx² + dy²)
    double dist = sqrt(pow(dx, 2) + pow(dy, 2));

    cout << "Distance: " << dist << endl;
    return 0;
}
Expected Output
Distance: 5

What it does: the classic 3-4-5 triangle. pow(dx, 2) squares each side, sqrt takes the root of the total.

For simple squaring, dx * dx is faster and clearer than pow(dx, 2).

Part B · The Second Trap

Storing a double Into an int Cuts the Fraction

Assignment does not round. It throws the fractional part away.

Cut, not rounded

int t = 3.99;
cout << t;      // 3

Nearly 4, but stored as 3.

Round on purpose

int t = round(3.99);
cout << t;      // 4

Say what you mean.

Why it matters: sqrt and pow return double. If a result is 4.999999 instead of 5, storing it in an int silently gives 4.

Safe habit: int n = (int)round(pow(a, b));

A Short Comparison

How Python Differs

Python has no separate char type — a single character is just a short string. In C++ a char is a number, which is why - '0' works.

Be Careful

Common Mistakes

  • char c = 5;
    gives ASCII code 5, not '5'; use '0' + 5
  • Adding characters
    '4' + '2' is 102, not "42"
  • Forgetting - '0'
    summing "4729" gives 214, not 22
  • "Age: " + age
    will not compile; wrap with to_string
  • int n = 3.99;
    stores 3; use round if you want 4
  • 10 / 3 for an average
    integer division gives 3; use 10.0 / 3

The habit: ask "is this value a number, or the text of a number?"

103

Both are numbers: 51 + 52 = 103. C++ adds their ASCII codes, it does not join them as text.

Solutions (4–6)

Exercise Answers

4 · Count even digits

string s = "90210";
int count = 0;
for (int i = 0; i < s.length(); i++) {
    int d = s[i] - '0';
    if (d % 2 == 0) count++;
}
cout << count << endl;   // 3

The digits are 9, 0, 2, 1, 0. The even ones are 0, 2, 0 → 3. Remember that 0 counts as even.

5 · Negative rounding

floor(-2.5) = -3 ceil(-2.5) = -2 round(-2.5) = -3

floor goes left, ceil goes right, round goes away from zero on a tie.

6 · sqrt(26)

sqrt(26) is about 5.099. Assigning to an int cuts the fraction, giving 5. To round on purpose: int x = round(sqrt(26));

Wrap Up

Summary

  • to_string(n) turns a number into text; stoi(s) and stod(s) turn text into numbers.
  • A char is a number — its ASCII code. '0' is 48.
  • Digit → character: '0' + d.   Character → digit: c - '0'.
  • <cmath> gives sqrt, pow, abs, floor, ceil, round, fmod.
  • floor goes left, ceil goes right — even for negative numbers.
  • Storing a double in an int cuts the fraction; use round to round.

Always ask: is this a number, or the text of a number?
