02 · The Idea

What Is Recursion?

Recursion is when a function solves a problem by calling itself on a smaller version of the same problem.

Everyday picture

Two mirrors facing each other show a smaller copy inside a smaller copy, again and again. Each copy is the same picture, just smaller.

In programming

To solve a big task, do one small step, then ask the same function to solve the slightly smaller rest — until the task is tiny enough to answer directly.

Key requirement: the problem must get smaller each time, and there must be a point where it simply stops.

03 · Every Recursion Has Two Parts

Base Case & Recursive Case

1 · Base case

The simplest version, answered directly with no further call. This is what stops the recursion.

Example: the factorial of 1 is 1.

2 · Recursive case

Do one step, then call the function again on a smaller input that moves toward the base case.

Example: n! = n × (n−1)!

int factorial(int n) {
    if (n <= 1) return 1;           // base case: stop here
    return n * factorial(n - 1);    // recursive case: smaller n
}

Rule: always write the base case first, or the function may never stop.

Worked Example 1

Countdown (recursive void)

#include <iostream>
using namespace std;

void countdown(int n) {
    if (n == 0) {              // base case
        cout << "Done!" << endl;
        return;                // stop
    }
    cout << n << " ";          // one step
    countdown(n - 1);          // smaller problem
}

int main() {
    countdown(3);
    return 0;
}
Expected Output
3 2 1 Done!

Trace: countdown(3) prints 3, then calls countdown(2) → prints 2, then countdown(1) → prints 1, then countdown(0) → prints "Done!" and stops.

Notice: each call uses a smaller n, heading toward the base case n == 0.

Worked Example 2

Factorial (returns a value)

#include <iostream>
using namespace std;

int factorial(int n) {
    if (n <= 1) return 1;            // base case
    return n * factorial(n - 1);     // recursive case
}

int main() {
    cout << factorial(3) << endl;    // 6
    cout << factorial(5) << endl;    // 120
    return 0;
}
Expected Output
6 120

What it does: factorial(n) returns n multiplied by the factorial of n − 1, until it reaches 1.

Next slide: let us see exactly how the calls stack up and return.

04 · Under the Hood

The Call Stack: factorial(3)

Each call waits for the smaller call to finish. Calls pile up going down, then values come back going up.

Worked Example 3

Sum From 1 to n

#include <iostream>
using namespace std;

int sumTo(int n) {
    if (n == 0) return 0;          // base case
    return n + sumTo(n - 1);       // recursive case
}

int main() {
    cout << sumTo(4) << endl;       // 4+3+2+1+0
    return 0;
}
Expected Output
10

What it does: sumTo(4) = 4 + sumTo(3) = 4 + 3 + sumTo(2) … down to sumTo(0) which returns 0.

Pattern: same shape as factorial — one step plus a smaller call, with a clear stopping point.

05 · Two Ways

Recursion vs. a Loop

Loop (iteration)

int factLoop(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++)
        result *= i;
    return result;
}

Recursion

int factRec(int n) {
    if (n <= 1) return 1;
    return n * factRec(n - 1);
}
  • Both give the same answer. Many problems can be written either way.
  • Loops are usually faster and use less memory.
  • Recursion can be shorter and clearer for problems that are naturally "smaller copies of themselves".
06 · Be Careful

Common Mistakes

  • No base case
    the function calls forever and the program crashes
  • Not getting smaller
    calling f(n) instead of f(n - 1) never reaches the base case
  • Base case after the call
    check the stop condition first

Stack overflow

int bad(int n) {
    return n + bad(n - 1);
    // no base case!
    // crashes: stack overflow
}

Each call uses a little memory. Endless calls run out of space and the program stops with an error.

07 · Side by Side

Python vs. C++ Recursion

Python

def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

C++

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

Takeaway: the recursive idea is identical — a base case and a smaller call. C++ only adds the return type, the parameter type, and braces.

09 · Solutions (1–2)

Exercise Answers

1 · Power

int power(int base, int exp) {
    if (exp == 0) return 1;          // base case
    return base * power(base, exp-1);
}
// power(2, 3) -> 8
power(2, 3)
8

2 · Print down

void printDown(int n) {
    if (n == 0) return;     // base case
    cout << n << " ";       // print first
    printDown(n - 1);       // then recurse
}
// printDown(4) -> 4 3 2 1
printDown(4)
4 3 2 1

Exercise Answers

3 · Print up

void printUp(int n) {
    if (n == 0) return;     // base case
    printUp(n - 1);         // recurse FIRST
    cout << n << " ";       // then print
}
// printUp(4) -> 1 2 3 4

Why: printing after the call reverses the order.

int countDigits(int n) {
    if (n < 10) return 1;            // base case

}
// countDigits(572) -> 3

5 · Fibonacci

int fib(int n) {
    if (n <= 1) return n;       // fib(0)=0, fib(1)=1
    return fib(n-1) + fib(n-2); // two calls
}
// fib(6) -> 8
fib(6)
8

6 · Multiply by adding

int multiply(int a, int b) {
    if (b == 0) return 0;          // base case
    return a + multiply(a, b - 1);
}
// multiply(4, 3) -> 12
multiply(4, 3)
12
10 · Wrap Up

Summary

  • Recursion is a function that calls itself on a smaller problem.
  • Every recursion needs a base case (to stop) and a recursive case (to shrink).
  • Write the base case first, and make sure the input gets smaller each call.
  • Calls pile up on the call stack, then values return back up.
  • Many problems can be solved by recursion or a loop; pick the clearer one.
  • No base case (or input that never shrinks) causes a stack overflow.

Well done — you now have the full toolkit: input/output, decisions, loops, arrays, strings, functions, and recursion.
