The Idea

What Is a Struct?

A struct groups several related values under one name. Instead of many loose variables, you keep them together as one record.

The Syntax

Defining and Using a Struct

// define the struct (a blueprint)
struct Point {
    int x;
    int y;
};

int main() {
    Point p;          // create one
    p.x = 3;          // set a field with a dot
    p.y = 5;
    cout << p.x << ", " << p.y;  // 3, 5
    return 0;
}
  • struct Point { ... }; defines the record.
  • The values inside are called fields (or members).
  • Reach a field with a dot: p.x.

Like a class, a struct definition ends with a semicolon: };

A Handy Shortcut

Initializing a Struct

struct Point {
    int x;
    int y;
};

int main() {
    // fill the fields in order, in one line
    Point p = {3, 5};

    cout << p.x << endl;   // 3
    cout << p.y << endl;   // 5
    return 0;
}
Expected Output
3 5

What it does: {3, 5} fills the fields in the order they were declared — x first, then y.

This is a quick way to set up a simple record.

Worked Example 1

A Point Record

#include <iostream>
using namespace std;

struct Point {
    int x;
    int y;
};

int main() {
    Point a = {1, 2};
    Point b;
    b.x = 10;
    b.y = 20;

    cout << a.x << "," << a.y << endl;  // 1,2
    cout << b.x << "," << b.y << endl;  // 10,20
    return 0;
}
Expected Output
1,2 10,20

What it does: two separate Point records, each with its own x and y. Changing b does not affect a.

More Than Data

Structs Can Have Methods Too

struct Rectangle {
    int width;
    int height;

    int area() {              // a method inside a struct
        return width * height;
    }
};
  • In C++, a struct can hold methods, not just data.
  • This makes a struct very close to a class.
  • The method uses the struct's own fields directly.

Note: in the older C language, a struct could only hold data. C++ extended it.

Worked Example 2

A Struct With Behaviour

#include <iostream>
using namespace std;

struct Rectangle {
    int width;
    int height;
    int area() {
        return width * height;
    }
};

int main() {
    Rectangle r = {4, 3};
    cout << r.area() << endl;   // 12
    return 0;
}
Expected Output
12

What it does: the struct stores a width and height, and its area() method multiplies them. We call it with a dot: r.area().

Many Records

Arrays of Structs

struct Student {
    string name;
    int id;
};

int main() {
    Student list[2];
    list[0].name = "Sara"; list[0].id = 101;
    list[1].name = "Ali";  list[1].id = 102;

    for (int i = 0; i < 2; i++)
        cout << list[i].name << " " << list[i].id << endl;
    return 0;
}
Expected Output
Sara 101 Ali 102

What it does: an array where each slot is a whole Student record. We reach a field with list[i].name.

Worked Example 3

A Function Using a Struct

#include <iostream>
using namespace std;

struct Rectangle {
    int width;
    int height;
};

int area(Rectangle r) {        // takes a whole struct
    return r.width * r.height;
}

int main() {
    Rectangle box = {5, 2};
    cout << area(box) << endl;   // 10
    return 0;
}
Expected Output
10

What it does: a whole struct is passed into a function as one value. Inside, we read its fields with a dot.

The Important Part

The One Real Difference

In C++, struct and class are almost the same. The only difference is the default access level: struct members are public by default; class members are private by default.

Side by Side

Struct vs. Class

Default accesspublicprivate
Can hold datayesyes
Can hold methodsyesyes
Can use public / privateyesyes

In short: everything a struct can do, a class can do. The only language difference is the default access.

A Practical Guide

Which One Should You Use?

Use a struct when…

  • You just want to group a few related values.
  • The data is simple and meant to be open.
  • Example: a Point, a Date, a Colour.

Use a class when…

  • You want to protect the data (encapsulation).
  • The object has real behaviour and rules.
  • Example: a BankAccount, a Student.

This is a convention, not a rule. The compiler treats them the same apart from the default access.

Be Careful

Common Mistakes

  • Missing the semicolon
    a struct ends with };
  • Wrong initializer order
    {3, 5} fills fields in declared order
  • Forgetting the dot
    use p.x, not px
  • Expecting privacy by default
    struct fields are public unless you say otherwise
  • Thinking structs hold only data
    in C++ they can have methods too
  • Too many braces
    { {..}, {..} } only for nested data

Remember: struct and class differ only in the default access level.

Solution (6)

Exercise Answer

struct S { int a; };   // a is PUBLIC by default
class  C { int a; };   // a is PRIVATE by default

int main() {
    S s; s.a = 1;      // OK: public
    C c;
    // c.a = 1;        // ERROR: a is private
}

Answer: the struct's field a is usable from outside, but the class's field a is not.

Why: struct members are public by default, while class members are private by default. That single rule is the only difference between them.

Wrap Up

Summary

  • A struct groups related values (fields) under one name.
  • Create a struct variable and reach its fields with a dot: p.x.
  • {a, b} fills the fields in their declared order.
  • In C++, a struct can have methods too — almost like a class.
  • The only difference: struct is public by default, class is private by default.
  • Use a struct for plain data records; use a class when you want to protect the data.
