02 · A New Idea — Built From Old Ones

From 1D to 2D

This topic was not in your Python course, so it is new. The good news: it is built from two things you already know in C++.

You know: a 1D array

A single row of boxes (Lecture 6).

int row[4];   // one line of 4 boxes

You know: nested loops

A loop inside a loop (Lecture 5).

for (rows)
    for (cols)  // visit each cell

Put them together: a 2D array is a grid of rows and columns, and nested loops are how we move through it.

03 · The Picture

A 2D Array Is a Grid

A two-dimensional array has rows and columns. Each cell is found by two indices: first the row, then the column. Both start at 0.

04 · The Syntax

Declaring and Filling a Grid

// 1. declare: 2 rows, 3 columns (empty)
int grid[2][3];

// 2. declare AND give values, row by row
int m[2][3] = {
    {1, 2, 3},   // row 0
    {4, 5, 6}    // row 1
};

// put a value into one cell
grid[0][1] = 9;   // row 0, column 1
  • The first number is the row count, the second is the column count.
  • Each inner { } fills one row.
  • Both counts must be known in advance (fixed size).

Read it as: "m is 2 rows of 3 integers each — 6 cells in total."

05 · Reaching a Cell

Accessing Elements

#include <iostream>
using namespace std;

int main() {
    int m[2][3] = {
        {1, 2, 3},
        {4, 5, 6}
    };

    cout << m[0][0] << endl;  // 1  (row 0, col 0)
    cout << m[1][2] << endl;  // 6  (row 1, col 2)

    m[0][2] = 99;            // change one cell
    cout << m[0][2] << endl;  // 99
    return 0;
}
Expected Output
1 6 99
  • Always write the row first, then the column.
  • Valid rows are 0 to (rows − 1); valid columns are 0 to (cols − 1).
06 · The Main Skill

Looping Over a Grid

int m[2][3] = {{1,2,3},{4,5,6}};

// outer loop = rows, inner loop = columns
for (int r = 0; r < 2; r++) {
    for (int c = 0; c < 3; c++) {
        cout << m[r][c] << " ";
    }
    cout << endl;   // new line after each row
}
Expected Output
1 2 3 4 5 6

The pattern: the outer loop picks a row; the inner loop visits every column in that row.

Key: the endl goes after the inner loop, so each row prints on its own line.

Worked Example 1

Read and Print a Grid

#include <iostream>
using namespace std;

int main() {
    int g[2][2];

    // read 4 numbers into the grid
    for (int r = 0; r < 2; r++)
        for (int c = 0; c < 2; c++)
            cin >> g[r][c];

    // print as a grid
    for (int r = 0; r < 2; r++) {
        for (int c = 0; c < 2; c++)
            cout << g[r][c] << " ";
        cout << endl;
    }
    return 0;
}
If user types 1 2 3 4
1 2 3 4

What it does: the first nested loop fills the grid in row order; the second prints it back as a neat table.

07 · Under the Hood

How a 2D Array Lives in Memory

Memory is really one long line. C++ stores a 2D array row by row — all of row 0, then all of row 1. This is called row-major order.

Worked Example 2

Total and Row Sums

#include <iostream>
using namespace std;

int main() {
    int m[2][3] = {{1,2,3},{4,5,6}};
    int total = 0;

    for (int r = 0; r < 2; r++) {
        int rowSum = 0;
        for (int c = 0; c < 3; c++) {
            rowSum += m[r][c];   // add this row
        }
        cout << "Row " << r << " sum: " << rowSum << endl;
        total += rowSum;
    }
    cout << "Total: " << total << endl;
    return 0;
}
Expected Output
Row 0 sum: 6 Row 1 sum: 15 Total: 21

What it does: the inner loop adds one row; the outer loop adds each row's total into the grand total.

Worked Example 3

Find the Largest Value

#include <iostream>
using namespace std;

int main() {
    int m[2][3] = {{4, 9, 2},{7, 1, 8}};
    int biggest = m[0][0];   // start with the first cell

    for (int r = 0; r < 2; r++)
        for (int c = 0; c < 3; c++)
            if (m[r][c] > biggest)
                biggest = m[r][c];

    cout << "Largest = " << biggest << endl;
    return 0;
}
Expected Output
Largest = 9

What it does: it assumes the first cell is the largest, then checks every cell and updates when it finds a bigger one.

Reuse: the same idea as the 1D "find the largest" — just with two loops.

08 · An Important Rule

Passing a 2D Array to a Function

#include <iostream>
using namespace std;

// the COLUMN count must be given; rows can be passed in
void printGrid(int g[][3], int rows) {
    for (int r = 0; r < rows; r++) {
        for (int c = 0; c < 3; c++)
            cout << g[r][c] << " ";
        cout << endl;
    }
}

int main() {
    int m[2][3] = {{1,2,3},{4,5,6}};
    printGrid(m, 2);
    return 0;
}
Expected Output
1 2 3 4 5 6

The rule: you may leave the row count empty, but you must give the column count: int g[][3].

Why: C++ needs the column count to find each cell in memory (row-major).

09 · Beyond Two

Three Dimensions and More

A 3D array

// 2 layers, 3 rows, 4 columns
int cube[2][3][4];

cube[0][1][2] = 5;   // three indices

You can add more dimensions. A 3D array is like a stack of grids (layers).

  • 2D needs two loops; 3D needs three nested loops.
  • Each extra dimension adds one index and one loop.

Advice: 2D covers most needs. Use 3D only when the data is truly three-dimensional (for example, layers of a grid).

10 · Be Careful

Common Mistakes

  • Swapping the indices
    m[col][row] instead of m[row][col]
  • Out of range
    for m[2][3], rows are 0–1 and columns are 0–2
  • endl in the wrong place
    inside the inner loop it breaks the line too often
  • Mismatched braces
    each row needs its own { } in the initializer
  • Missing column count
    a function parameter needs int g[][cols]
  • Mixing up the counts
    the first size is rows, the second is columns
11 · A Familiar Comparison

Like a Python List of Lists

Python (list of lists)

m = [[1, 2, 3],
     [4, 5, 6]]
print(m[1][2])   # 6

C++ (2D array)

int m[2][3] = {{1,2,3},
               {4,5,6}};
cout << m[1][2];  // 6

The access m[row][col] is the same in both. The difference: a C++ 2D array is a fixed rectangle of one type, while a Python list of lists can grow and can have rows of different lengths.

14 · Wrap Up

Summary

  • A 2D array is a grid with rows and columns: int m[rows][cols].
  • Access a cell with m[row][col] — row first, both starting at 0.
  • Use nested loops (outer = rows, inner = columns) to fill, read, and print.
  • In memory, a 2D array is stored row by row (row-major order).
  • When passing to a function, the column count is required: int g[][cols].
  • More dimensions add one index and one loop each — but 2D covers most needs.

Grids are everywhere: tables, images, game boards, and spreadsheets all use 2D arrays.
