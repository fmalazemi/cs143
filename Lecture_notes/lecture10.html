Why Pointers

Every Variable Has an Address

In Lecture 6 you saw that variables live at numbered addresses in memory. A pointer is a variable that stores such an address. That single idea unlocks three things.

What pointers let you do

  • Let a function change the caller's variable.
  • Walk through arrays efficiently.
  • Use dynamic memory (ask for space at run time).

Part A · Pointers

What a Pointer Looks Like

A pointer is a box that stores the address of another box. Here p holds the address of x, so "p points to x".

p does not hold 42 — it holds 1000, the address where 42 is stored.

Part A · Pointers

Address-of & and Declaring a Pointer

#include <iostream>
using namespace std;

int main() {
    int x = 42;

    // &x is "the address of x"
    int *p = &x;        // p points to x

    cout << x  << endl;  // 42       (the value)
    cout << &x << endl;  // address  (e.g. 0x...)
    cout << p  << endl;  // same address
    return 0;
}
  • int *p; declares p as a pointer to an int. The * marks it as a pointer.
  • &x gives the address of x.
  • So p = &x stores x's address in p.

Read it as: "p is a pointer that holds the address of x."

Part A · Pointers

Dereference with *

Putting * before a pointer means "go to that address and use the value there". This is dereferencing.

p is the address; *p is the value at that address.

Worked Example 1 · Pointers

Reading and Changing Through a Pointer

#include <iostream>
using namespace std;

int main() {
    int x = 42;
    int *p = &x;        // p points to x

    cout << *p << endl;  // 42  (value via pointer)

    *p = 100;           // change x THROUGH the pointer
    cout << x << endl;   // 100 (x itself changed)
    return 0;
}
Expected Output
42 100

What it does: *p reads the value of x. Then *p = 100 writes through the pointer, so x itself becomes 100.

Key: the pointer and the variable share the same box.

Part A · Pointers

Null Pointers

#include <iostream>
using namespace std;

int main() {
    int *p = nullptr;   // points to nothing (safe value)

    if (p != nullptr) {
        cout << *p << endl;   // only use it if valid
    } else {
        cout << "p points to nothing" << endl;
    }
    return 0;
}
Expected Output
p points to nothing
  • nullptr is a special "points to nothing" value.
  • Use it to mark a pointer that is not ready yet.
  • Never dereference a null pointer — it crashes the program.

Habit: check if (p != nullptr) before using *p.

Part A · Pointers

Changing a Caller's Variable

#include <iostream>
using namespace std;

// receives the ADDRESS of a number
void addOne(int *p) {
    *p = *p + 1;     // change the original
}

int main() {
    int n = 5;
    addOne(&n);      // pass the address of n
    cout << n << endl;  // 6
    return 0;
}
Expected Output
6

What it does: the function gets the address of n, so changing *p changes n back in main.

Compare: this does the same job as a reference (Lecture 7). References are simpler; pointers are more flexible.

Worked Example 2 · Pointers

Swap Using Pointers

#include <iostream>
using namespace std;

void swap(int *a, int *b) {
    int temp = *a;   // save the value a points to
    *a = *b;         // a's box gets b's value
    *b = temp;       // b's box gets the saved value
}

int main() {
    int x = 1, y = 2;
    swap(&x, &y);    // pass both addresses
    cout << x << " " << y << endl;
    return 0;
}
Expected Output
2 1

What it does: the function swaps the values through the two pointers, so the change is seen in main.

Notice: we pass &x and &y, and inside we work with *a and *b.

Part A · Pointers

Pointers and Arrays

An array's name is really the address of its first element. So a pointer can step through an array.

Worked Example 3 · Pointers

Walking an Array With a Pointer

#include <iostream>
using namespace std;

int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;        // points to arr[0]

    for (int i = 0; i < 3; i++) {
        cout << *(p + i) << " ";  // value at p + i
    }
    cout << endl;
    return 0;
}
Expected Output
10 20 30

What it does: p + i moves to the i-th element, and *(p + i) reads its value.

Note: p[i] means the same as *(p + i) — array indexing is pointer math underneath.

Part B

Static & Dynamic Memory

Now that you can hold an address, let us see the two places data lives: the stack (static) and the heap (dynamic).

Part B · Memory

Two Kinds of Memory

A running C++ program uses memory in two main areas. They behave very differently.

Static — the Stack

  • Holds your local variables.
  • Size fixed when you write the code.
  • Created and freed for you; fast but limited.

Dynamic — the Heap

  • Memory you request with new.
  • Size and lifetime chosen at run time.
  • Large and flexible — but you must free it.

Python note: Python manages all of this automatically. C++ gives you the stack for free, but the heap is your responsibility.

Part B · Memory

The Memory Map

A local pointer p lives on the stack, but the value it points to (from new) lives on the heap.

Part B · Memory

Static: Local Variables on the Stack

#include <iostream>
using namespace std;

void f() {
    int x = 5;        // x is created on the stack
    int arr[3];       // also on the stack
    cout << x << endl;
}                     // x and arr are freed automatically here

int main() {
    f();
    return 0;
}
  • Local variables are placed on the stack automatically.
  • They disappear when their { } block ends — you do nothing.
  • Each function call gets its own stack frame (recursion, Lecture 8).

Limit: the stack is small. A huge local array or very deep recursion causes a stack overflow.

Part B · Memory

Dynamic: new and delete

#include <iostream>
using namespace std;

int main() {
    int *p = new int;   // reserve one int on the heap

    *p = 50;            // use it through the pointer
    cout << *p << endl;  // 50

    delete p;           // give the memory back
    return 0;
}
Expected Output
50
  • new reserves heap memory and returns its address.
  • You reach the value through a pointer (*p).
  • delete frees it when you are done.

Rule: every new needs exactly one matching delete.

Worked Example 4 · Memory

Stack vs. Heap Lifetime

#include <iostream>
using namespace std;

int main() {
    // stack: lives only inside this block
    {
        int s = 10;
        cout << s << endl;   // 10
    }   // s is gone here

    // heap: lives until we delete it
    int *h = new int;
    *h = 20;
    cout << *h << endl;       // 20
    delete h;                // we choose when it ends
    return 0;
}
Expected Output
10 20

The difference: the stack variable s ends automatically at the closing brace. The heap value lives until you call delete.

Power and duty: the heap lets you control lifetime — but you must remember to free it.

Part B · Memory

Dynamic Arrays

A fixed array needs a size known when you write the code. A heap array can have a size chosen while the program runs.

Worked Example 5 · Memory

An Array Sized by the User

#include <iostream>
using namespace std;

int main() {
    int n;
    cout << "How many? ";
    cin >> n;

    int *a = new int[n];     // size decided at run time!

    for (int i = 0; i < n; i++)
        a[i] = i * i;        // 0, 1, 4, 9, ...

    for (int i = 0; i < n; i++)
        cout << a[i] << " ";
    cout << endl;

    delete[] a;              // free an ARRAY with delete[]
    return 0;
}
If user types 5
How many? 5 0 1 4 9 16

Why this matters: a normal array cannot do this — its size must be fixed in the code. The heap lets the size come from input.

Note: free an array with delete[], not delete.

Part B · Memory

Data That Outlives a Function

Wrong — returns a dead address

int* makeNumber() {
    int x = 7;     // on the stack
    return &x;     // x dies when function ends!
}   // the returned address is now invalid

Right — heap survives

int* makeNumber() {
    int *p = new int;  // on the heap
    *p = 7;
    return p;          // still valid after return
}   // caller must delete it later

Key idea: a stack variable dies when its function ends. To hand data back from a function, put it on the heap — but the caller becomes responsible for the delete.

Worked Example 6 · Memory

A Function That Builds an Array

#include <iostream>
using namespace std;

// build a heap array of the first n squares
int* makeSquares(int n) {
    int *a = new int[n];
    for (int i = 0; i < n; i++)
        a[i] = (i + 1) * (i + 1);
    return a;            // heap data survives the return
}

int main() {
    int *sq = makeSquares(4);
    for (int i = 0; i < 4; i++)
        cout << sq[i] << " ";
    cout << endl;
    delete[] sq;         // caller frees it
    return 0;
}
Expected Output
1 4 9 16

What it does: the function reserves an array on the heap, fills it, and returns the pointer. The data stays alive after the function ends.

Contract: whoever receives the pointer must delete[] it.

Be Careful · Pointers

Common Pointer Mistakes

  • Uninitialized (wild) pointer
    int *p; *p = 5; writes to a random place
  • Dereferencing null
    *p when p is nullptr crashes
  • Forgetting &
    int *p = x; needs &x
  • Confusing *
    in a declaration it marks a pointer; in code it dereferences
  • Dangling pointer
    pointing to a local variable that has ended
  • Wrong target type
    an int* must point to an int

Safe habit: start a pointer at nullptr, and check before you dereference.

Be Careful · Memory

Common Memory Mistakes

  • Memory leak
    new without a matching delete wastes memory
  • Double delete
    calling delete p twice crashes
  • Wrong delete form
    free arrays with delete[], single values with delete
  • Leak in a loop
    new inside a loop with no delete adds up fast
  • Using after delete
    the pointer is now dangling
  • Huge stack array
    very large local arrays overflow the stack

Safe habit: right after writing new, write the matching delete before you forget.

Wrap Up

Summary

  • A pointer stores the address of another variable; &x gives an address, *p the value.
  • Pointers let a function change the caller's variable and let you walk arrays.
  • Static memory (the stack) holds local variables, freed automatically with their scope.
  • Dynamic memory (the heap) is requested with new and released with delete.
  • A dynamic array (new int[n]) lets the size be chosen at run time; free it with delete[].
  • Start pointers at nullptr, never dereference null, and match every new with a delete.

Pointers plus dynamic memory are the foundation for the data structures you will meet next.
