02 · Why Pointers

Every Variable Has an Address

In Lecture 6 you saw that array boxes live at numbered addresses in memory. Every variable has one. A pointer is simply a variable that stores such an address.

You have already met this idea

  • Arrays sit at addresses in memory (Lecture 6).
  • References let a function change the caller's variable (Lecture 7).

Pointers go one step further

They let you hold an address in a variable, pass it around, and reach the value it points to. This powers arrays, function output, and dynamic memory.

Python note: Python hides addresses from you; C++ lets you work with them directly.

03 · The Picture

What a Pointer Looks Like

A pointer is a box that stores the address of another box. Here p holds the address of x, so we say "p points to x".

p does not hold 42 — it holds 1000, the address where 42 is stored.

04 · Two Operators

Address-of & and Declaring a Pointer

#include <iostream>
using namespace std;

int main() {
    int x = 42;

    // &x is "the address of x"
    int *p = &x;        // p points to x

    cout << x  << endl;  // 42       (the value)
    cout << &x << endl;  // address  (e.g. 0x...)
    cout << p  << endl;  // same address
    return 0;
}
  • int *p; declares p as a pointer to an int. The * marks it as a pointer.
  • &x gives the address of x.
  • So p = &x stores x's address in p.

Read it as: "p is a pointer that holds the address of x."

05 · Following the Pointer

Dereference with *

Putting * before a pointer means "go to that address and use the value there". This is called dereferencing.

p is the address; *p is the value at that address.

Worked Example 1

Reading and Changing Through a Pointer

#include <iostream>
using namespace std;

int main() {
    int x = 42;
    int *p = &x;        // p points to x

    cout << *p << endl;  // 42  (value via pointer)

    *p = 100;           // change x THROUGH the pointer
    cout << x << endl;   // 100 (x itself changed)
    return 0;
}
Expected Output
42 100

What it does: *p reads the value of x. Then *p = 100 writes through the pointer, so x itself becomes 100.

Key: the pointer and the variable share the same box.

06 · Pointing at Nothing

Null Pointers

#include <iostream>
using namespace std;

int main() {
    int *p = nullptr;   // points to nothing (safe value)

    if (p != nullptr) {
        cout << *p << endl;   // only use it if valid
    } else {
        cout << "p points to nothing" << endl;
    }
    return 0;
}
Expected Output
p points to nothing
  • nullptr is a special "points to nothing" value.
  • Use it to mark a pointer that is not ready yet.
  • Never dereference a null pointer — it crashes the program.

Habit: check if (p != nullptr) before using *p.

07 · Pointers Into Functions

Changing a Caller's Variable

#include <iostream>
using namespace std;

// receives the ADDRESS of a number
void addOne(int *p) {
    *p = *p + 1;     // change the original
}

int main() {
    int n = 5;
    addOne(&n);      // pass the address of n
    cout << n << endl;  // 6
    return 0;
}
Expected Output
6

What it does: the function gets the address of n, so changing *p changes n back in main.

Compare: this does the same job as a reference (Lecture 7). References are simpler; pointers are more flexible.

Worked Example 2

Swap Using Pointers

#include <iostream>
using namespace std;

void swap(int *a, int *b) {
    int temp = *a;   // save the value a points to
    *a = *b;         // a's box gets b's value
    *b = temp;       // b's box gets the saved value
}

int main() {
    int x = 1, y = 2;
    swap(&x, &y);    // pass both addresses
    cout << x << " " << y << endl;
    return 0;
}
Expected Output
2 1

What it does: the function swaps the values through the two pointers, so the change is seen in main.

Notice: we pass &x and &y, and inside we work with *a and *b.

08 · A Close Relationship

Pointers and Arrays

An array's name is really the address of its first element. So a pointer can step through an array.

Worked Example 3

Walking an Array With a Pointer

#include <iostream>
using namespace std;

int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;        // points to arr[0]

    for (int i = 0; i < 3; i++) {
        cout << *(p + i) << " ";  // value at p + i
    }
    cout << endl;
    return 0;
}
Expected Output
10 20 30

What it does: p + i moves to the i-th element, and *(p + i) reads its value.

Note: p[i] means the same as *(p + i) — array indexing is pointer math underneath.

09 · Dynamic Memory (a First Look)

new and delete

#include <iostream>
using namespace std;

int main() {
    // ask for one int while the program runs
    int *p = new int;

    *p = 50;                 // use it through the pointer
    cout << *p << endl;       // 50

    delete p;                // give the memory back
    return 0;
}
Expected Output
50
  • new reserves memory and returns its address.
  • delete frees that memory when you are done.
  • Rule: every new needs a matching delete.

Why: forgetting delete wastes memory (a "memory leak").

10 · Two Similar Tools

Pointers vs. References

FeatureReference (&)Pointer (*)
What it isanother name for a variablea variable holding an address
Must be set at startyes, alwaysno (but should be)

Guideline: prefer references when you simply need to share a variable; use pointers when you need to repoint, allow "nothing", or manage dynamic memory.

11 · Be Careful

Common Mistakes

  • Uninitialized (wild) pointer
    int *p; *p = 5; writes to a random place
  • Dereferencing null
    *p when p is nullptr crashes
  • Forgetting &
    int *p = x; needs &x
  • Confusing *
    in a declaration it marks a pointer; in code it dereferences
  • Dangling pointer
    pointing to a local variable that has ended
  • Leak
    using new without a matching delete

Safe habit: start a pointer at nullptr, and check before you dereference.

Python vs. C++

Takeaway: Python manages memory and addresses for you. C++ gives you direct control — powerful, but you must be careful.

15 · Wrap Up

Summary

  • A pointer is a variable that stores the address of another variable.
  • &x gives an address; *p gives the value at an address (dereferencing).
  • int *p = &x; makes p point to x; *p = 100; changes x.
  • Pass a pointer to let a function change the caller's variable.
  • An array name is the address of its first element, so pointers can walk arrays.
  • Start pointers at nullptr, never dereference null, and match every new with a delete.

Pointers are the foundation for dynamic data structures you will meet next.
