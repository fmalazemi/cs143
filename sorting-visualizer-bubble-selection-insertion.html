      How the three compare
      

        
All three are quadratic — but not for the same reason

        
Selection sort always makes n(n−1)/2 comparisons, no matter what the data looks like. Bubble and insertion sort adapt: on data that is already nearly sorted they finish in close to n steps. Try the “Nearly sorted” shape on all three and watch the comparison counter.

        
Movement is the real difference

        
Selection sort makes at most n−1 swaps — the fewest of the three — because it only writes once per pass. Bubble sort can swap on almost every comparison. Insertion sort shifts rather than swaps, which is one write per moved element instead of three.

        
Which one would you actually use

        
Insertion sort. It is fast on small and nearly sorted inputs, it is stable, and it is what real libraries fall back to for tiny sub-arrays inside quicksort or merge sort. Bubble sort survives only as a teaching example.

        
What they all share

        
Each pass places at least one value permanently, and each pass costs a scan. n passes × n work = n². Merge sort escapes this by making the passes cost log n instead of n.
